# Architecture Composition

**Where we are.** A single transformer block — multi-head attention, a position-wise feed-forward network, layer normalization, and a residual connection — is by itself a shallow map. This chapter composes blocks into the architectures that are actually trained: we stack $N$ blocks into an encoder and a decoder, introduce **cross-attention** as the one new operation that couples them, organize the three standard architectures (encoder-only, encoder-decoder, decoder-only) by their design choices, state what depth and composition buy through the universal-approximation theorem, and count the per-layer cost that makes attention the quadratic bottleneck. We write $\operatorname{MHA}$, $\operatorname{FFN}$, and $\operatorname{LN}$ for the block's sublayers and $\operatorname{Attn}(\mathbf Q,\mathbf K,\mathbf V)$ for scaled dot-product attention, all fixed in earlier chapters.

## The encoder

The starting picture is the simplest composition: feed the input through one block, then feed that output through another, $N$ times, with every block free to look at every position. Nothing is masked, so each layer produces a representation of one token informed by the whole sequence, and the stack deepens that contextualization.

<Theorem kind="definition" name="Transformer encoder" id="def-encoder">
A **transformer encoder** is a stack of $N$ identical pre-norm blocks with **bidirectional** self-attention (no causal mask). Writing $\operatorname{Block}^{(\ell)}$ for the $\ell$-th block with its own learned parameters,
$$
\mathbf H^{(0)} = \mathbf X, \qquad
\mathbf H^{(\ell)} = \operatorname{Block}^{(\ell)}\!\big(\mathbf H^{(\ell-1)}\big),
\qquad \ell = 1,\dots,N,
$$
where $\mathbf X\in\R^{n\times d}$ is the input sequence of $n$ token representations. The output $\mathbf H^{(N)}\in\R^{n\times d}$ is a contextualized representation of the **entire** input: each row depends on every input position.
</Theorem>

<MarginNote variant="pattern" label="Encoder">Bidirectional context — every token sees every other token. The pattern behind BERT <Cite key="devlin2019bert" /> and the encoder half of T5 <Cite key="raffel2020t5" />.</MarginNote>

The recursion is just function composition, $\mathbf H^{(N)} = \operatorname{Block}^{(N)}\circ\cdots\circ\operatorname{Block}^{(1)}(\mathbf X)$, but the consequence is structural: because no block masks any position, the map $\mathbf X\mapsto\mathbf H^{(N)}$ is bidirectional throughout. We will see in the next section that masking exactly one of the matrix products inside the block converts this into the causal map a decoder needs.

<Theorem kind="remark" name="The Vision Transformer is this encoder" id="rem-vit">
Feed the encoder of <XRef id="def-encoder" /> the patch embeddings of <XRef id="def-patch-embedding" /> together with a positional encoding over the patch index — the original model used a *learned 1D* table over the raster order, with axial 2D schemes (<XRef id="def-pe-2d" />) an alternative it found no better — prepending a learned $\texttt{[CLS]}$ slot (<XRef id="def-pooled-rep" />). The result is the **Vision Transformer** <Cite key="dosovitskiy2021vit" />: bidirectional self-attention over a patch grid, classified from the pooled slot exactly as the encoder-only classifier of <XRef id="def-encoder-classifier" />. Not one line of the block, the stack, or the universal-approximation argument changes — only $\mathbf X$ is sourced from patches rather than tokens. The modality lives entirely in the embedding interface of <XRef id="def-embedding-interface" />.
</Theorem>

## Cross-attention and the decoder

A decoder must do two things an encoder need not: respect causality (it generates one token at a time and may not peek at the future) and condition on a *separate* input sequence (in translation, the source sentence). The first is handled by the causal mask already developed for self-attention (<XRef id="def-causal-mask" />). The second needs one genuinely new operation. Cross-attention is the same scaled dot-product attention as before, with one asymmetry: the queries come from the sequence being generated, while the keys and values come from a fixed, already-encoded sequence.

<Theorem kind="definition" name="Cross-attention" id="def-cross-attention">
Let $\mathbf X_{\text{dec}}\in\R^{m\times d}$ be a decoder sequence and $\mathbf H_{\text{enc}}\in\R^{n\times d}$ an encoder output. **Cross-attention** projects queries from the decoder and keys and values from the encoder:
$$
\begin{aligned}
\mathbf Q&\defeq\mathbf X_{\text{dec}}\mathbf W^Q\in\R^{m\times d_k},
&\mathbf W^Q&\in\R^{d\times d_k},\\
\mathbf K&\defeq\mathbf H_{\text{enc}}\mathbf W^K\in\R^{n\times d_k},
&\mathbf W^K&\in\R^{d\times d_k},\\
\mathbf V&\defeq\mathbf H_{\text{enc}}\mathbf W^V\in\R^{n\times d_v},
&\mathbf W^V&\in\R^{d\times d_v},\\
\mathbf C&\defeq\softmax\!\left(\frac{\mathbf Q\mathbf K^\top}{\sqrt{d_k}}\right)\mathbf V\in\R^{m\times d_v},
&\mathbf W^O&\in\R^{d_v\times d},\\
\operatorname{CrossAttn}(\mathbf X_{\text{dec}},\mathbf H_{\text{enc}})
&\defeq\mathbf C\mathbf W^O\in\R^{m\times d}. &&
\end{aligned}
$$
Each decoder query thereby attends over **all** $n$ encoder positions, and the explicit output projection restores model width so the result is type-compatible with a residual addition. Multi-head cross-attention applies these projections per head, concatenates the $m\times d_v$ head contexts, and uses $\mathbf W^O\in\R^{h d_v\times d}$ exactly as <XRef id="def-mha" />.
</Theorem>

<MarginNote variant="formula" label="Q vs K, V">Queries from the decoder, keys and values from the encoder — the only structural difference from self-attention, where all three come from one sequence.</MarginNote>

Because cross-attention uses scaled dot-product attention, each weight row is row-stochastic: every row of the pre-projection context $\mathbf C$ is a convex combination of the encoder value rows (<XRef id="prop-row-stochastic-convex" />). After $\mathbf W^O$, the residual-width output is equivalently a convex combination of the **projected** value rows $\mathbf v_j\mathbf W^O$. The attention core selects and averages encoded source features; the learned linear output projection then remixes their coordinates. With this operation in hand, the decoder block has three sublayers: the encoder block's self-attention *masked*, the one genuinely new sublayer — cross-attention — and the feed-forward network.

<Theorem kind="remark" name="Vision-language fusion is cross-attention" id="rem-multimodal-crossattn">
The "separate input sequence" the decoder conditions on need not be text. Let $\mathbf H_{\text{img}}\in\R^{n_v\times d}$ be patch features from a Vision Transformer (<XRef id="rem-vit" />) and $\mathbf X_{\text{txt}}\in\R^{m\times d}$ a text stream. Conditioning text generation on the image is exactly <XRef id="def-cross-attention" /> with the encoder stream taken to be vision:
$$
\operatorname{CrossAttn}(\mathbf X_{\text{txt}},\mathbf H_{\text{img}})
= \softmax\!\left(
\frac{(\mathbf X_{\text{txt}}\mathbf W^Q)(\mathbf H_{\text{img}}\mathbf W^K)^\top}{\sqrt{d_k}}
\right)(\mathbf H_{\text{img}}\mathbf W^V)\mathbf W^O.
$$
The convexity guarantee applies before the output projection: each pre-projection fused text position is a convex combination of the image value rows, and after $\mathbf W^O$ it is a convex combination of their projected rows. This select-and-average core is cross-attention fusion; later residual and FFN sublayers can move outside that convex hull. Flamingo's *gated* variant <Cite key="alayrac2022flamingo" /> refines it twice — the vision features first pass through a learned-query resampler (<XRef id="def-resampler" />), and the cross-attention enters through a $\tanh$ gate that starts closed (<XRef id="def-gated-xattn" />).
</Theorem>

<Theorem kind="definition" name="Transformer decoder block" id="def-decoder-block">
A **decoder block** applies three sublayers in order, each with its own pre-norm and residual connection:
$$
\begin{aligned}
\mathbf Z_1 &= \mathbf H + \operatorname{MHA}^{\text{causal}}\!\big(\operatorname{LN}(\mathbf H)\big) && \text{(masked self-attention),}\\
\mathbf Z_2 &= \mathbf Z_1 + \underbrace{\operatorname{CrossAttn}\!\big(\operatorname{LN}(\mathbf Z_1),\,\mathbf H_{\text{enc}}\big)}_{\in\,\R^{m\times d}\text{ after }\mathbf W^O} && \text{(cross-attention),}\\
\operatorname{Block}(\mathbf H) &= \mathbf Z_2 + \operatorname{FFN}\!\big(\operatorname{LN}(\mathbf Z_2)\big) && \text{(feed-forward).}
\end{aligned}
$$
The masked self-attention lets position $i$ attend only to decoder positions $j\le i$; the cross-attention lets it attend to all encoder positions; the FFN is applied position-wise.
</Theorem>

<MarginNote variant="pattern" label="Decoder block">Three sublayers, not two: masked self-attention, then cross-attention, then FFN — each pre-normed and residual.</MarginNote>

Stacking the encoder of <XRef id="def-encoder" /> against a stack of these blocks gives the original architecture.

<Theorem kind="definition" name="Encoder-decoder transformer" id="def-enc-dec">
An **encoder-decoder transformer** <Cite key="vaswani2017attention" /> pairs:
<Sidenote>This is the architecture of the original "Attention Is All You Need," and of sequence-to-sequence models such as T5 <Cite key="raffel2020t5" />.</Sidenote>
- an **encoder** (<XRef id="def-encoder" />) that maps the input sequence to $\mathbf H_{\text{enc}} = \mathbf H^{(N)}$, computed **once**; and
- a **decoder**, a stack of $N$ decoder blocks (<XRef id="def-decoder-block" />) sharing that single $\mathbf H_{\text{enc}}$, that generates output tokens autoregressively, each step conditioned on $\mathbf H_{\text{enc}}$ and on all previously generated tokens.
</Theorem>

When the task does not have a separate input to condition on — language modeling, where the model continues its own text — the encoder and the cross-attention it feeds are both unnecessary. Removing them leaves a single causal stack.

<Theorem kind="definition" name="Decoder-only transformer" id="def-decoder-only">
A **decoder-only transformer** is a single stack of $N$ blocks with causal self-attention and **no** cross-attention. Input and output share one sequence: given tokens $x_1,\dots,x_{t-1}$, the model predicts the next token $x_t$. This is the architecture of GPT <Cite key="radford2018improving" /> and LLaMA <Cite key="touvron2023llama" />, and of most modern large language models.
</Theorem>

<MarginNote variant="tip" label="Conditioning">A decoder-only model has no separate encoder stream, so demonstrations and queries can condition one another inside the causal prompt. The next chapter constructs what attention can compute from that shared context (<XRef id="def-in-context-learning" />).</MarginNote>

<Theorem kind="remark" name="The three architectures as design choices" id="rem-three-archs">
The three standard encoder/generator architectures are three settings of two binary switches — is the output stack's self-attention bidirectional or causal, and does it have cross-attention to a separate source?

| Architecture | Self-attention | Cross-attention | Typical use |
|---|---|---|---|
| Encoder-only | bidirectional | none | classification, embeddings (BERT <Cite key="devlin2019bert" />) |
| Encoder-decoder | causal decoder + bidirectional encoder | decoder → encoder | sequence-to-sequence (T5 <Cite key="raffel2020t5" />, the original transformer <Cite key="vaswani2017attention" />) |
| Decoder-only | causal | none | autoregressive generation (GPT, LLaMA <Cite key="touvron2023llama" />) |

The table names the three standard *representation or autoregressive-generation* patterns, not every usable cell of the switch space. A bidirectional output stack with cross-attention is perfectly valid for two-stream fusion or reranking; it is non-autoregressive because its output rows can read future output positions. This is distinct from the usual **cross-encoder**, which concatenates both inputs into one bidirectional self-attention stack and therefore occupies the bidirectional/no-cross-attention cell. The encoder–decoder-families chapter abstracts the causal/bidirectional and conditioning choices to arbitrary sequence mixers (<XRef id="def-patterns" />) and instantiates the same three standard patterns across attention, recurrent, and state-space families.

An encoder-only model typically adds a learned summary position (a $\texttt{[CLS]}$ token) and trains a small head on that position's final hidden state. It is itself an encoder despite having no cross-attention. In an encoder–decoder model, a *separate source encoder* feeds the decoder through cross-attention; a concatenation-based multimodal model may likewise use an upstream vision encoder even though its language stack has no cross-attention.
</Theorem>

<Theorem kind="remark" name="Two fusion routes are two switch settings" id="rem-unified-sequence">
A generative vision-language model couples the two modalities by one of two routes, corresponding to the two settings of the language stack's cross-attention switch in <XRef id="rem-three-archs" />. **Cross-attention present:** a decoder attends to a separate image stream (<XRef id="rem-multimodal-crossattn" />) — the encoder-decoder pattern with a vision encoder in place of a text one. **Cross-attention absent:** project the image features — in practice a frozen vision encoder's output features, as in LLaVA, not raw patch embeddings — into the token space and **concatenate** them with the text tokens into one sequence $\mathbf X = [\,\mathbf X_{\text{img}};\,\mathbf X_{\text{txt}}\,]$, processed by a single decoder-only stack (<XRef id="def-decoder-only" />) — the embedding interface of <XRef id="def-embedding-interface" /> used twice, with no new operation <Cite key="liu2023llava" />. Thus cross-attention decides *how the already-encoded visual features enter the language stack*, not whether an upstream encoder exists. Both routes are built out in full later: the projector and learned-query resampler that fill the concatenation route (<XRef id="def-connector" />), and the single interleaved stream that subsumes both into one decoder (<XRef id="def-interleaved" />).
</Theorem>

## Universal approximation

A composition of blocks is only worth stacking if depth and width buy representational power. The classical universal-approximation theorems concern a single-hidden-layer network approximating a continuous function on a compact set; they say nothing about *sequence-to-sequence* maps or about the specific attention-plus-FFN architecture. The theorem below supplies the missing statement, and — more useful for intuition — its proof strategy says *why* the architecture is expressive: attention routes information across positions while the FFN stores function values, so together they implement an approximate quantize-route-lookup pipeline.

<Theorem kind="theorem" name="Universal approximation for transformers" id="thm-universal-approx">
Fix $n\ge2$, $d\ge1$, and $1\le p<\infty$. For a measurable domain $D\subseteq\R^{n\times d}$, define
$$
d_{p,D}(f,g)
\defeq
\left(\int_D \norm{f(\mathbf X)-g(\mathbf X)}_p^p\,d\mathbf X\right)^{1/p},
$$
where $\norm{\cdot}_p$ is the entrywise matrix $\ell^p$ norm and $d\mathbf X$ is Lebesgue measure. Let $\mathcal T^{2,1,4}$ be the class of arbitrary finite-depth compositions of blocks, each block applying in order:

- residual dot-product self-attention sublayers with exactly two heads, scalar query/key/value width per head, and a learned output projection back to width $d$; and
- residual position-wise two-layer ReLU FFNs of hidden width $4$,

with learned FFN biases and query-projection biases, and **no layer normalization**. Then:<Sidenote>These fixed-width $L^p$ density results are Theorems 2 and 3 of <Cite key="yun2020transformers" />. They are representation theorems for a deliberately idealized residual attention-plus-ReLU class, not for the normalized pre-norm block defined earlier.</Sidenote>

1. **Without positional encoding.** For every continuous, compactly supported map $f:\R^{n\times d}\to\R^{n\times d}$ satisfying
   $f(\mathbf P\mathbf X)=\mathbf P f(\mathbf X)$ for every $n\times n$ permutation matrix $\mathbf P$, and every $\epsilon>0$, there is $g\in\mathcal T^{2,1,4}$ with $d_{p,\R^{n\times d}}(f,g)<\epsilon$.
2. **With learned additive positional encoding.** For every compact domain $K\subset\R^{n\times d}$ with $0<\lambda(K)<\infty$, every continuous $f:K\to\R^{n\times d}$, and every $\epsilon>0$, there are a learned table $\mathbf E\in\R^{n\times d}$ and $g\in\mathcal T^{2,1,4}$ such that
   $d_{p,K}\big(f,\mathbf X\mapsto g(\mathbf X+\mathbf E)\big)<\epsilon$.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the sketch: plain self-attention treats the input as a *set* (permuting tokens permutes outputs). Which of the two statements must that symmetry force, and what single ingredient breaks it?</MarginNote>

<Theorem kind="proof" name="Proof strategy (sketch)">
We do not reproduce the full construction; we name its three stages and the hypotheses that make each legitimate.

**Quantize by FFNs.** Continuity and compact support (statement 1), or compactness of $K$ (statement 2), let a sufficiently fine cubical partition replace $f$ by a piecewise-constant map with arbitrarily small $L^p$ error. A stack of residual width-$4$ ReLU FFNs realizes an approximation to this coordinatewise quantizer; the construction need only fail on thin cell boundaries, which is why the metric is $L^p$ rather than uniform.

**Route by self-attention.** On the resulting finite grid, the two scalar-width attention heads build a contextual mapping: each row receives a code depending on the whole quantized sequence, equal inputs related by a row permutation receive correspondingly permuted codes, and inputs not related by a permutation receive distinct codes. This is the route step.

**Look up by FFNs; separate positions when needed.** Further position-wise ReLU layers map each contextual code to the required grid value. Without position, every constituent operation commutes with $\mathbf X\mapsto\mathbf P\mathbf X$, giving statement 1's equivariant class. For statement 2, after an affine rescaling of the compact input range into $[0,1]^{n\times d}$ (absorbed into learned affine maps), one may choose the additive table
$$
\mathbf E_{t,:}=2(t-1)\mathbf1,\qquad t=1,\dots,n.
$$
Row $t$ then lies in the coordinate band $[2(t-1),\,2(t-1)+1]^d$, disjoint from every other row's band. The same quantize-route-lookup construction can therefore identify absolute row positions and implement an arbitrary continuous target in $L^p$. The cited paper supplies the layer-by-layer construction and error bounds. $\qquad\blacksquare$
</Theorem>

<Theorem kind="remark" name="Why depth helps: iterative refinement" id="rem-depth">
The construction needs contextual coordinates, and one layer provides exactly one round of context-mixing (attention) followed by one round of pointwise processing (FFN). Stacking layers therefore builds those coordinates progressively: a later layer can route over summaries that earlier layers already computed. The analogy to numerical iteration is precise — each layer is one step of a message-passing scheme that refines the representation, and <Cite key="geshkovski2023mathematical" /> make it literal by reading the layers as discrete steps of a gradient flow on an interacting-particle system.
</Theorem>

<Theorem kind="remark" name="Existence is not optimization" id="rem-existence">
The theorem is **existential**: it guarantees that a finite, potentially very deep transformer with the stated fixed internal widths — two scalar-width heads and a width-$4$ ReLU FFN — *exists* to approximate each fixed-length target. It does not apply layer normalization, it does not provide a useful finite-depth bound for practical model selection, and it is silent on whether gradient descent will find those parameters or how the solution generalizes off the approximation domain. Expressivity, optimization, and generalization are separate questions; only the first is settled here.
</Theorem>

## Computational complexity

Expressivity says depth is worth having; cost says where the bill is paid. We count, per layer, the time and the activation memory of the two dominant operations — self-attention and the FFN — and locate the sequence length at which the quadratic attention term overtakes the linear-in-$n$ FFN term.

<Theorem kind="proposition" name="Per-layer complexity" id="prop-complexity">
For sequence length $n$, model dimension $d$, $h$ heads of query/key width $d_k$ and value width $d_v$, and feed-forward hidden dimension $d_{\text{ff}}$, a dense multi-head block has the following leading time and materialized-activation memory costs:

| Component | Time | Memory |
|---|---|---|
| Q/K/V and output projections | $\Theta\!\big(n h d(d_k+d_v)\big)$ | $\Theta\!\big(nh(d_k+d_v)+nd\big)$ |
| Scores and softmax, $\mathbf Q\mathbf K^\top$ | $\Theta(hn^2d_k)$ | $\Theta(hn^2)$ |
| Weight–value products, $\mathbf A\mathbf V$ | $\Theta(hn^2d_v)$ | $\Theta(hnd_v)$ beyond the stored weights |
| Feed-forward network | $\Theta(n d d_{\text{ff}})$ | $\Theta(nd_{\text{ff}}+nd)$ |
| **Total per layer** | $\Theta\!\big(nhd(d_k+d_v)+hn^2(d_k+d_v)+ndd_{\text{ff}}\big)$ | $\Theta\!\big(hn^2+hn(d_k+d_v)+nd_{\text{ff}}+nd\big)$ |

The **quadratic mixing term** overtakes the FFN at sequence scale
$$
n=\Theta\!\left(\frac{d\,d_{\text{ff}}}{h(d_k+d_v)}\right)
$$
and overtakes the linear attention projections at $n=\Theta(d)$. Thus it is the block's asymptotic time bottleneck beyond the larger of those two scales. These statements concern the quadratic term, not the sum of every attention projection, and assume dense materialization; <XRef id="rem-flash-attention" /> changes the memory schedule.
</Theorem>

<MarginNote variant="formula" label="Crossover">With $h\mid d$ and $d_k=d_v=d/h$, the two attention mixes and the two FFN maps have equal leading scalar-multiplication counts at $n=d_{\text{ff}}$. Projection work is linear in $n$ and is included in the total above.</MarginNote>

<Theorem kind="proof">
Count each matrix product at its displayed general width; constants are retained here only to locate the conventional crossover.

**Projections.** Per head, forming $\mathbf Q$ and $\mathbf K$ costs $2ndd_k$ scalar multiplications and forming $\mathbf V$ costs $ndd_v$. Across heads, the concatenated $n\times hd_v$ context times $\mathbf W^O\in\R^{hd_v\times d}$ costs another $nhd_vd$. The total is $2nhd(d_k+d_v)$, hence $\Theta(nhd(d_k+d_v))$ time, and the projected activations occupy $\Theta(nh(d_k+d_v)+nd)$ memory.

**Attention mixing.** Each head's $\mathbf Q\mathbf K^\top$ costs $n^2d_k$ and its $\mathbf A\mathbf V$ costs $n^2d_v$, for $hn^2(d_k+d_v)$ across heads. Materializing one attention-weight array costs $\Theta(hn^2)$; projected values and head contexts add $\Theta(hnd_v)$ and are already represented in the total's linear terms.

**Feed-forward.** The two position-wise maps cost $2ndd_{\text{ff}}$ scalar multiplications and store the $n\times d_{\text{ff}}$ hidden activation plus residual-width input/output, giving the table's FFN row.

**Crossover.** Equating quadratic mixing with the two FFN products gives
$$
hn^2(d_k+d_v)=2ndd_{\text{ff}}
\quad\Longleftrightarrow\quad
n=\frac{2d\,d_{\text{ff}}}{h(d_k+d_v)}.
$$
Equating mixing with all four attention projections gives $n=2d$. With the standard assumption $h\mid d$ and choice $d_k=d_v=d/h$, the first equality reduces to $n=d_{\text{ff}}$ and the second to $n=2d$. Hence for the common $d_{\text{ff}}=4d$, the quadratic mixing term becomes the largest of these components at $n>d_{\text{ff}}=4d$; total attention time, which also includes projections, can cross the FFN earlier. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="tip" label="Memory wall">The $O(n^2)$ attention memory is the long-sequence bottleneck: at $n = 100{,}000$ a single attention matrix in 32-bit floats already needs $\sim 40$ GB.</MarginNote>

The complexity table treats the attention map as a fixed cost, but the two terms can be attacked separately — one by changing the operator, one by changing only the algorithm.

<NotationOverride scope="Linear-attention derivation">
The feature vectors $\phi(\mathbf k_j)$ and values $\mathbf v_j$ are **columns** here, so $\phi(\mathbf k_j)\mathbf v_j^\top$ is a $D\times d_v$ outer product. Outputs return to row form.
</NotationOverride>

<Theorem kind="remark" name="Linear attention changes the operator" id="rem-linear-attention">
If the softmax kernel is replaced by an explicit feature map $\phi:\R^{d_k}\to\R^{D}$, the output at query $i$ becomes
$$
\text{output}_i = \frac{\phi(\mathbf q_i)^\top\sum_{j}\phi(\mathbf k_j)\mathbf v_j^\top}{\phi(\mathbf q_i)^\top\sum_{j}\phi(\mathbf k_j)}.
$$
The matrix $\sum_j\phi(\mathbf k_j)\mathbf v_j^\top\in\R^{D\times d_v}$ is computed **once** and shared across queries, so each output costs $O(D d_v)$ and the total is $O(n D d_v)$ — *linear* in $n$ <Cite key="katharopoulos2020transformers" />. This is a different function: with the standard choice $\phi(x) = \operatorname{elu}(x)+1 > 0$ the weights remain non-negative and row-normalized, so convexity survives; what is given up is the exponential kernel's temperature-controlled sharpness, trading approximation quality for the asymptotic gain. In the causal case the shared sums become *running* sums — a state updated token by token — and that observation is upgraded to a theorem later: causal linear attention **is** a matrix-state recurrence (<XRef id="thm-linattn-rnn" />), and with decay it is exactly a selective state space model (<XRef id="thm-ssd" />).
</Theorem>

<Theorem kind="remark" name="FlashAttention changes only the algorithm" id="rem-flash-attention">
The $O(n^2)$ **memory** of materializing the attention matrix can be removed without altering the result. FlashAttention <Cite key="dao2022flashattention" /> tiles the computation into blocks that fit in fast on-chip memory and accumulates the softmax in a numerically stable online pass, never storing the full $n\times n$ matrix; the additional activation memory drops from quadratic to linear in $n$ at fixed head dimension. The output is **exact** — the function computed is identical to scaled dot-product attention. This is an algorithmic optimization, not a mathematical one.
</Theorem>

Three ways to pay the attention bill, and only one of them changes the function computed:

| Method | Operator | Output | Time / layer | Activation memory |
|---|---|---|---|---|
| Softmax attention | exponential kernel | exact | $O(n^2 d)$ | $O(n^2)$ |
| FlashAttention | the *same* operator | exact — identical function | $O(n^2 d)$ | $O(n)$ |
| Linear attention | a *different* operator | approximate | $O(n D d_v)$ — linear in $n$ | $O(n)$ |

FlashAttention (<XRef id="rem-flash-attention" />) is an *algorithm*: it tiles the same computation to cut memory, leaving both the function and the $O(n^2 d)$ time unchanged. Linear attention (<XRef id="rem-linear-attention" />) is a *different operator*: swapping the exponential kernel for a feature map $\phi$ buys linear time and memory, at the cost of exactness.

## A concrete parameter count

The asymptotics above hide constants that matter in practice. We count the parameters of one standard block explicitly, then assemble a full model, and read off the dominant scaling term — the quantity that governs how parameter count grows with width and depth.

<Theorem kind="proposition" name="Dominant block parameter count" id="prop-params">
Assume $h\mid d$. For a pre-norm block with $d_k = d_v = d/h$ and feed-forward width $d_{\text{ff}}$, the multi-head attention contributes $4d^2$ weight parameters: each of the three per-head projection families has $d^2$ entries in total across heads, and the output projection has $d^2$. The FFN contributes $2 d\, d_{\text{ff}}$ from its two dense maps. Biases and layer-norm scales and shifts are lower order. Hence one block has dominant count
$$
4d^2 + 2 d\, d_{\text{ff}},
$$
and with $d_{\text{ff}} = 4d$ this is $4d^2 + 8d^2 = 12 d^2$ per block, so $N$ blocks contribute $\approx 12 N d^2$.
</Theorem>

<Theorem kind="proof">
**Proof.**

For queries, each of the $h$ heads has a matrix in $\R^{d\times(d/h)}$, so the family contains $h\,d(d/h)=d^2$ weights; keys and values contribute the same count, for $3d^2$ total. Concatenation has width $h(d/h)=d$, so $\mathbf W^O\in\R^{d\times d}$ adds $d^2$. The FFN matrices have shapes $d\times d_{\text{ff}}$ and $d_{\text{ff}}\times d$, adding $2dd_{\text{ff}}$. Therefore the dominant weight count is $4d^2+2dd_{\text{ff}}$; substituting $d_{\text{ff}}=4d$ and multiplying by $N$ gives $12Nd^2$. $\qquad\blacksquare$
</Theorem>

<Theorem kind="example" name="Parameter count for a concrete model" id="ex-param-count">
Take $d=512$, $h=8$, $d_k=d_v=64$, $d_{\text{ff}}=2048$, $N=6$, $|\mathcal V| = 32{,}000$.

**Embedding.** $\mathbf E\in\R^{|\mathcal V|\times d}$: $32{,}000\times 512 = 16{,}384{,}000 \approx 16.4\text{M}$.

**Per block** (<XRef id="prop-params" />). MHA $4d^2 = 4\cdot 512^2 = 1{,}048{,}576$; FFN $2 d\, d_{\text{ff}} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$; two layer-norms $2\cdot 2d = 2{,}048$. Block total $\approx 3{,}147{,}776 \approx 3.15\text{M}$.

**Model.** Embedding $16.4\text{M}$, plus $N=6$ blocks at $3.15\text{M}$ giving $18.9\text{M}$, plus an output projection of $0$ when tied to the embedding. **Total $\approx 35.3\text{M}$ parameters.**

**Forward FLOPs at $n=1024$.** Count one multiply-add as two FLOPs. The Q/K/V/output projections cost $8nd^2=8\cdot1024\cdot512^2\approx2.15\times10^9$ FLOPs. The two quadratic attention products (<XRef id="prop-complexity" />) cost $2n^2d$ each — $\mathbf Q\mathbf K^\top$ and $\mathbf A\mathbf V$ — totalling $4n^2d\approx2.15\times10^9$. The two FFN maps cost $4ndd_{\text{ff}}\approx4.29\times10^9$. Thus the complete block costs about $8.59\times10^9$ FLOPs and the six blocks about $51.5$ GFLOPs. Here the FFN is about half of total block FLOPs; the quadratic mixing term alone is about one quarter. Since $n=1024<d_{\text{ff}}=2048$, that quadratic term has not yet overtaken the FFN, although projection work means **total** attention already matches it in this example. The $O(n^2)$ materialized-memory pressure persists at every length.
</Theorem>

## Exercises

Attempt each before expanding its solution.

**Exercise 7.1 (Prove).** Show that the pre-projection context in <XRef id="def-cross-attention" /> is, in every row, a convex combination of the encoder value rows, and that the residual-width output is a convex combination of their rows after $\mathbf W^O$.

<SolutionBox>

Write $\mathbf A=\softmax(\mathbf Q\mathbf K^\top/\sqrt{d_k})$. Each row is row-stochastic, so pre-projection context row $i$ is $\mathbf c_i=\sum_jA_{ij}\mathbf v_j$, a convex combination of the encoder value rows (<XRef id="prop-row-stochastic-convex" />). Linearity of the output projection gives
$$
\mathbf c_i\mathbf W^O=\sum_jA_{ij}(\mathbf v_j\mathbf W^O),
$$
with the same nonnegative coefficients summing to one. Thus the cross-attention output lies in the convex hull of the projected value rows. Later residual and FFN operations need not remain in that hull. $\qquad\blacksquare$

</SolutionBox>

**Exercise 7.2 (Construct).** Specify the two binary switches — self-attention causal vs bidirectional, cross-attention present vs absent — and exhibit the setting that yields each of the encoder-only, encoder-decoder, and decoder-only architectures.

<SolutionBox>

The switches are (a) output-stack self-attention causal vs bidirectional and (b) cross-attention present vs absent. **Encoder-only:** bidirectional, no cross-attention (BERT). **Encoder-decoder:** bidirectional source encoder + causal decoder with cross-attention pulling $\mathbf K,\mathbf V$ from $\mathbf H_{\text{enc}}$ (original transformer, T5). **Decoder-only:** causal, no cross-attention; prompt tokens occupy the same stream and can supply demonstrations before the query, the setting formalized by <XRef id="def-in-context-learning" /> (GPT, LLaMA). The fourth setting, bidirectional + cross-attention, is valid for fusion or reranking but is not a fourth autoregressive architecture: bidirectional output self-attention can read future output positions, so the stack is not prefix-consistent. $\qquad\blacksquare$

</SolutionBox>

**Exercise 7.3 (Compute).** For $d = 1024$, $d_{\text{ff}} = 4096$, $h\mid d$, and the standard $d_k=d_v=d/h$, find the sequence length at which the **quadratic attention-mixing term** overtakes the FFN in leading matrix-multiplication work, and explain why FlashAttention does not move that asymptotic crossover.

<SolutionBox>

Across heads, the two quadratic products require $2n^2d$ scalar multiplications, while the two FFN maps require $2ndd_{\text{ff}}$. Equating them and cancelling $2nd>0$ gives $n=d_{\text{ff}}=4096$. This is the crossover of the quadratic mixing term with the FFN; Q/K/V/output projections are additional work linear in $n$, so total attention can cross earlier. FlashAttention does not move the stated asymptotic crossover: it computes the same quadratic operator while changing the tiling and materialized-memory schedule. $\qquad\blacksquare$

</SolutionBox>

**Exercise 7.4 (Compute).** Using $4d^2 + 2 d\, d_{\text{ff}}$ per block with $d_{\text{ff}} = 4d$, derive the $\approx 12 N d^2$ block-parameter scaling, and state which model component the embedding term $|\mathcal V|\, d$ instead governs.

<SolutionBox>

Per block, MHA contributes $4d^2$ and the FFN $2 d\,d_{\text{ff}}$ (layer-norm scale/shift are $O(d)$, lower order), so one block is $4d^2 + 2 d\,d_{\text{ff}}$. With $d_{\text{ff}} = 4d$ this is $4d^2 + 8d^2 = 12d^2$, and $N$ blocks give $\approx 12 N d^2$. The term $|\mathcal V|\,d$ is separate: it governs the embedding (and tied output projection), scaling with vocabulary rather than depth. $\qquad\blacksquare$

</SolutionBox>

**Exercise 7.5 (Derive).** Explain why the no-position class in <XRef id="thm-universal-approx" /> is permutation-equivariant. Using the theorem's global $L^p$ metric, prove that an $L^p$ limit of such maps is equivariant almost everywhere, and give a continuous compactly supported target excluded by that closure.

<SolutionBox>

Let $\mathbf P$ be a permutation matrix. Shared row-wise projections send $\mathbf X\mapsto\mathbf P\mathbf X$ to $\mathbf Q,\mathbf K,\mathbf V\mapsto\mathbf P\mathbf Q,\mathbf P\mathbf K,\mathbf P\mathbf V$; logits become $\mathbf P(\mathbf Q\mathbf K^\top)\mathbf P^\top$, row-wise softmax transforms the same way, and residual attention and row-wise FFNs therefore satisfy $g(\mathbf P\mathbf X)=\mathbf P g(\mathbf X)$.

For an equivariant $g$, define the equivariance defect of a target $f$ at $\mathbf P$ by $\Delta_{\mathbf P}f(\mathbf X)=f(\mathbf P\mathbf X)-\mathbf P f(\mathbf X)$. The triangle inequality, equivariance of $g$, invariance of Lebesgue measure under $\mathbf X\mapsto\mathbf P\mathbf X$, and invariance of the entrywise norm under row permutations give
$$
\begin{aligned}
\norm{\Delta_{\mathbf P}f}_{L^p}
&\le \norm{f(\mathbf P\,\cdot)-g(\mathbf P\,\cdot)}_{L^p}
  +\norm{\mathbf P(g-f)(\cdot)}_{L^p}\\
&=2\,d_{p,\R^{n\times d}}(f,g).
\end{aligned}
$$
Hence if equivariant $g_r\to f$ in global $L^p$, then $\Delta_{\mathbf P}f=0$ almost everywhere. If $f$ is continuous, its defect is continuous, so equality almost everywhere implies equality everywhere.

For a counterexample, choose a continuous cutoff $\eta:[0,\infty)\to[0,1]$ with $\eta(r)=1$ for $r\le1$ and $\eta(r)=0$ for $r\ge2$, and define
$$
f(\mathbf X)\defeq \eta(\norm{\mathbf X}_F)
\begin{pmatrix}\mathbf x_1\\ \vdots\\ \mathbf x_1\end{pmatrix}.
$$
This map is continuous and compactly supported. At any $\mathbf X$ of Frobenius norm below $1$ with $\mathbf x_1\ne\mathbf x_2$, swapping rows $1$ and $2$ changes the broadcast source from $\mathbf x_1$ to $\mathbf x_2$, while permuting the already-broadcast output does nothing. Its equivariance defect is therefore nonzero on an open set, so the displayed bound rules out arbitrarily accurate no-position $L^p$ approximation. Learned positional encodings break precisely this symmetry. $\qquad\blacksquare$

</SolutionBox>
