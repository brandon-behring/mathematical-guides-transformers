# Sparse & Sub-Quadratic Attention

**Where we are.** Dense attention compares every query row with every key row. Its attention core therefore contributes the quadratic term in <XRef id="prop-complexity" />. Part 4 changed how that computation is trained, stored, and streamed: FlashAttention evaluates the same dense map without materializing it (<XRef id="thm-online-softmax-exact" />), while mixture-of-experts conditionally selects feed-forward parameters (<XRef id="def-moe" />). This chapter opens Part 5 by changing the sequence mixer itself. **Structural sparse attention usually keeps the score–normalize–average recipe but chooses which query-key pairs exist; distributional sparsity instead changes the normalizer.** The next chapter takes the other route: linear attention and selective state spaces replace the pairwise softmax operator with a recurrent state.

The distinction is mathematical, not merely terminological:

| Route around the quadratic wall | Scores evaluated | Exact dense softmax map? | Destination |
|---|---:|---:|---|
| IO-aware dense attention | all $\Theta(n^2)$ | yes | FlashAttention, <XRef id="thm-online-softmax-exact" /> |
| computationally sparse attention | a chosen subset | generally no | this chapter |
| kernel/state-space replacement | no dense score matrix | no | <XRef id="thm-linattn-rnn" />, <XRef id="def-selective-ssm" /> |

All definitions remain over rows $\mathbf x_i\in\R^d$. A row may represent a word, an image patch, an audio frame, or any other sequence unit; only the geometry used to choose its neighbors changes.

## Attention as a sparse edge set

The useful unit of accounting is not the number of zeros eventually written into an attention matrix. It is the number of dot products actually evaluated.

<Theorem kind="definition" name="Computationally sparse attention" id="def-sparse-attention">
For query rows $i=1,\dots,n$, let the nonempty set $\mathcal E_i\subseteq\{1,\dots,n\}$ contain the key rows that query $i$ is allowed to read. Define the directed **attention edge set**, row degree, and total retained-pair count
$$
\mathcal E\defeq\{(i,j):j\in\mathcal E_i\},
\qquad
r_i\defeq|\mathcal E_i|,
\qquad
m_{\mathcal E}\defeq|\mathcal E|=\sum_{i=1}^{n}r_i.
\tag{1}
$$
Equivalently, define $M^{\mathcal E}_{ij}=0$ on $\mathcal E$ and $-\infty$ off $\mathcal E$, then use the masked-logit formula of <XRef id="def-causal-mask" /> with $\mathbf M^{\mathcal E}$ in place of the causal mask.

The attention is **computationally sparse** only when its implementation enumerates the pairs in $\mathcal E$ without first forming the omitted entries of $\mathbf Q\mathbf K^\top$. A dense matrix multiplication followed by $M^{\mathcal E}$ represents the same mathematical map but is not a sparse computation.
</Theorem>

<MarginNote variant="warning" label="A mask is not a kernel">Writing many $-\infty$ entries proves which weights are zero. It does not prove that the corresponding dot products were skipped. Complexity follows $m_{\mathcal E}$ only under an edge-enumerating sparse kernel.</MarginNote>

<Theorem kind="proposition" name="Sparse-attention work is controlled by retained edges" id="prop-sparse-edge-complexity">
Suppose an implementation of <XRef id="def-sparse-attention" /> evaluates exactly the pairs in $\mathcal E$. For one head with key width $d_k$ and value width $d_v$, excluding the unchanged linear projections:

1. score formation costs $\Theta(m_{\mathcal E}d_k)$ arithmetic;
2. applying the normalized weights to values costs $\Theta(m_{\mathcal E}d_v)$ arithmetic;
3. materializing the sparse weights and their indices costs $\Theta(m_{\mathcal E})$ storage.

If every row has $r_i\le r_{\max}$, the attention core is $O(nr_{\max}(d_k+d_v))$ time and $O(nr_{\max})$ sparse-weight storage. Dense causal attention has $m_{\mathcal E}=n(n+1)/2$; hence an edge budget $r_{\max}=o(n)$ is sub-quadratic in sequence length.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof: write one dot product and one weighted value contribution for every retained pair $(i,j)$. Which loop count changes when the mask becomes sparse, and which projection costs do not?</MarginNote>

<Theorem kind="proof">
**Scores.** Each retained pair computes $\mathbf q_i^\top\mathbf k_j$, a $d_k$-coordinate inner product. Exactly $m_{\mathcal E}$ such pairs therefore cost $\Theta(m_{\mathcal E}d_k)$.

**Values.** After rowwise normalization over $\mathcal E_i$, each retained scalar multiplies and accumulates one $d_v$-vector, costing $\Theta(d_v)$ per pair and $\Theta(m_{\mathcal E}d_v)$ in total.

**Storage and specializations.** One stored scalar weight and one index per edge give $\Theta(m_{\mathcal E})$. If $r_i\le r_{\max}$, then $m_{\mathcal E}=\sum_i r_i\le nr_{\max}$. Under the causal mask, row $i$ has $i$ keys, so $m_{\mathcal E}=\sum_{i=1}^{n}i=n(n+1)/2$. Dividing the sparse upper bound $nr_{\max}$ by $n^2$ shows that $r_{\max}=o(n)$ makes the attention core sub-quadratic. The $ndd_k$ and $ndd_v$ projection terms remain because every row still produces a query, key, and value. $\qquad\blacksquare$
</Theorem>

## Static and factorized patterns

The architect can choose $\mathcal E$ before seeing the input. Static patterns are predictable, cheap to index, and often regular enough for block kernels. Their limitation is equally clear: content cannot create an edge that the pattern omitted.

### Sliding windows

The most important static pattern is also the bridge to the hybrid architectures later in Part 5. We preserve the causal, one-sided convention used by those decoders: $w$ includes the current token.

<Theorem kind="definition" name="Sliding-window attention" id="def-swa">
For a **positive-integer window size** $w\in\N$, **causal sliding-window attention** uses
$$
\mathcal E_i^{(w)}
\defeq
\{\max(1,i-w+1),\dots,i\},
$$
or, equivalently, the mask
$$
M^{(w)}_{ij}
\defeq
\begin{cases}
0, & 0\le i-j<w,\\[2pt]
-\infty, & \text{otherwise.}
\end{cases}
\tag{2}
$$
At $w\ge n$ this is the causal mask of <XRef id="def-causal-mask" />; at $w=1$ each position reads only itself.
</Theorem>

<Theorem kind="proposition" name="Sliding-window edge count and decode cache" id="prop-swa-edge-cache">
Let $w_*\defeq\min(n,w)$. Causal sliding-window attention has the exact edge count
$$
m_{\mathrm{SWA}}
=\sum_{i=1}^{n}\min(i,w)
=nw_*-\frac{w_*(w_*-1)}2.
\tag{3}
$$
Its sparse attention core therefore costs $\Theta(m_{\mathrm{SWA}}(d_k+d_v))$, which is $\Theta(nw(d_k+d_v))$ when $w\le n$. After decoding $t$ positions, a layer need retain at most $\min(t,w)d_{\text{cache}}$ cached elements. The steady-state cap is $w d_{\text{cache}}$, or $2wd$ for standard MHA where $d_{\text{cache}}=2d$ (<XRef id="prop-kv-cache" />).
</Theorem>

<MarginNote variant="tip" label="Self-explanation">How many rows are still ramping toward width $w$, and when can a cached key never enter another future window?</MarginNote>

<Theorem kind="proof">
**Edges.** Row $i$ retains $\min(i,w)$ keys. Splitting the triangular prefix from the constant-width rows gives
$$
\sum_{i=1}^{n}\min(i,w)
=\sum_{i=1}^{w_*-1}i+(n-w_*+1)w_*
=nw_*-\frac{w_*(w_*-1)}2.
$$
The work bound follows from <XRef id="prop-sparse-edge-complexity" />.

**Cache.** Once a key is more than $w-1$ positions behind the newest query, it belongs to no future sliding window. Retaining only the most recent $\min(t,w)$ key/value rows is therefore sufficient, and multiplying their per-row width by $d_{\text{cache}}$ gives the cap. This is the quantity used by the hybrid accounting in <XRef id="prop-decode-state" />. $\qquad\blacksquare$
</Theorem>

Locality in one layer does not mean locality forever. Information can hop through intermediate rows as layers compose.

<Theorem kind="proposition" name="Finite-depth receptive field of causal sliding windows" id="prop-swa-receptive-field">
In a depth-$L$ stack whose only cross-position operation is causal sliding-window attention of width $w$, the structural receptive field of output row $i$ is exactly
$$
\mathcal R_L(i)
=\{\max(1,i-L(w-1)),\dots,i\}.
\tag{4}
$$
Its size is $\min(i,1+L(w-1))\le\min(n,1+L(w-1))$. Thus fixed-width local reach grows linearly with depth until it saturates at the available prefix.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof: one layer can move information backward by any distance from $0$ to $w-1$. What distances can a path of $L$ such moves realize, and where does the beginning of the sequence truncate them?</MarginNote>

<Theorem kind="proof">
**Base case.** For $L=0$, row $i$ depends only on itself, matching (4).

**Inductive step.** Suppose depth $L-1$ reaches every row from $\max(1,j-(L-1)(w-1))$ through $j$ at an intermediate position $j$. The last layer lets $i$ read each $j\in\{\max(1,i-w+1),\dots,i\}$. The earliest possible source is therefore
$$
\max(1,i-w+1-(L-1)(w-1))
=\max(1,i-L(w-1)),
$$
and causality forbids anything after $i$.

**Exactness.** Conversely, every integer displacement $0\le q\le L(w-1)$ is a sum of $L$ integers in $\{0,\dots,w-1\}$: repeatedly take at most $w-1$ until the remainder is zero. Those layerwise edges give a path from $i-q$ to $i$ whenever $i-q\ge1$. Hence the interval is exact, and counting its rows gives the stated capped size. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="cross-ref" label="Patches use the same count">On an $H\times W$ patch grid, a two-dimensional radius-$a$ window retains at most $(2a+1)^2$ keys per query. Equation (1) immediately gives $O(HW(2a+1)^2(d_k+d_v))$ attention-core work. The rows changed geometry, not algebra.</MarginNote>

### Dilation, blocks, strides, and global tokens

A window can be widened without adding edges by spacing its samples. A sequence can instead be partitioned into blocks so whole tiles are admitted or rejected. A few global rows can provide shortcuts. The next proposition makes the counts explicit; it does not assert that every pattern preserves the same information.

<Theorem kind="proposition" name="Edge counts for common static sparse patterns" id="prop-static-sparse-counts">
Let $w,\Delta_{\mathrm{dil}},b,\rho$ be positive integers and $g$ a nonnegative integer. Under an implementation that enumerates only retained edges:

1. a causal **dilated window** $\mathcal E_i=\{i-q\Delta_{\mathrm{dil}}:0\le q<w,\ i-q\Delta_{\mathrm{dil}}\ge1\}$ has at most $nw$ edges and one-layer span at most $(w-1)\Delta_{\mathrm{dil}}$;
2. a **local-plus-strided factorization** with at most $b$ local keys and $\lceil n/b\rceil$ strided keys per row has at most $n(b+\lceil n/b\rceil)$ edges, minimized in order at $b\asymp\sqrt n$, giving $O(n^{3/2})$ edges;
3. after partitioning into blocks of at most $b$ rows, if each query block reads at most $\rho$ key blocks, **block-sparse attention** has at most $\lceil n/b\rceil\rho b^2=O(n\rho b+\rho b^2)$ edges;
4. designating $g$ of the final $n$ rows as bidirectional **global rows** adds at most $2gn$ directed edges to any base pattern, so constant $g$ preserves a linear-edge base order.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Count keys per query, then query blocks. Which $b$ balances $b+n/b$, and how many directed edges can one global row add?</MarginNote>

<Theorem kind="proof">
**Dilated and factorized.** A dilated row lists at most $w$ integers, so summing degrees gives at most $nw$; its earliest listed index is $i-(w-1)\Delta_{\mathrm{dil}}$. The factorized row lists at most $b+\lceil n/b\rceil$ keys. Summing over $n$ rows gives the bound, while AM-GM gives $b+n/b\ge2\sqrt n$ and the choice $b=\lceil\sqrt n\rceil$ attains the same order.

**Blocks and globals.** There are $\lceil n/b\rceil$ query blocks. Each of at most $\rho$ admitted key blocks contributes no more than $b^2$ query-key pairs, yielding the block bound. Each global row can send at most $n$ edges and receive at most $n$ edges, so $g$ of them add no more than $2gn$. $\qquad\blacksquare$
</Theorem>

Sparse Transformer instantiated the local-plus-strided idea and reported the $O(n\sqrt n)$ factorization <Cite key="child2019sparse" />. Longformer combined local windows, optional dilation, and task-chosen global rows for an encoder whose attention scales linearly at fixed budgets <Cite key="beltagy2020longformer" />. BigBird combined window, random, and global edges, leaving $O(n)$ retained pairs when the window and random degrees and the number of global rows are fixed <Cite key="zaheer2020bigbird" />. These names describe graph families, not one interchangeable mask: a causal decoder cannot let an early global row read future tokens, whereas a bidirectional document encoder can.

### What sparsity can preserve in principle

Fewer edges need not mean a smaller representable function class, but the theorem needs its own hypotheses. It is not inherited automatically from dense attention.

<Theorem kind="remark" name="Cited fixed-length sparse universality with a global-star graph" id="rem-bigbird-universality">
Fix sequence length $n$, width $d$, and $1<p<\infty$. Let $f:[0,1]^{n\times d}\to\R^{n\times d}$ be continuous. Consider a sparse Transformer with positional information and an auxiliary global row whose attention graph contains the star in which the global row reads every sequence row and every sequence row reads itself and the global row. Then, for every $\varepsilon>0$, the BigBird construction gives a sparse Transformer $g$ such that
$$
\left(\int_{[0,1]^{n\times d}}
\norm{f(\mathbf X)-g(\mathbf X)}_p^p\,d\mathbf X\right)^{1/p}
\le\varepsilon.
\tag{5}
$$
Choosing the star graph itself gives $O(n)$ edges per attention layer <Cite key="zaheer2020bigbird" />.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof sketch: if one global row can gather from every input and every row can read it back, where could a code for the entire fixed-length sequence live?</MarginNote>

**Proof strategy of the cited construction.** First approximate $f$ in $L^p$ by a piecewise-constant map on a sufficiently fine grid. Next, use sparse attention through the global row to construct a distinct contextual code for each grid input and output position: information is gathered into the global row and then made available to every ordinary row. Position-wise feed-forward maps send each contextual code to the required constant output. Finally, replace the construction's idealized hard selection and piecewise maps by softmax and ReLU approximations, allocating the error across the three stages to remain below $\varepsilon$. The star contributes only $O(n)$ edges. The detailed grid, code-separation, and error estimates are in the cited BigBird proof; random edges are not required for this universality construction.

This result is **parallel** to the dense result <XRef id="thm-universal-approx" />, not a refinement or corollary. It is fixed-length, $L^p$, noncausal, and uses a special global row. It says neither that one sparse layer equals one dense layer nor that every sparse mask is universal. A broader theorem gives sufficient conditions for sparse universality: self-loops, a Hamiltonian path in the union of cycling patterns, finite multi-hop reach from every row to every other row, and a probability map that can approach hardmax. Under those conditions, sparse Transformers with only $O(n)$ connections per layer can approximate the same fixed-length continuous function class <Cite key="yun2020oconnections" />. The connectivity hypotheses are the substance; the edge count alone is not.

## Content-routed and distributional sparsity

Static masks ask *where positions are*. Content routing asks *what they contain*. Reformer hashes queries and keys into locality-sensitive buckets and reports $O(n\log n)$ attention complexity <Cite key="kitaev2020reformer" />. Routing Transformer instead uses online clustering so that queries read keys assigned to the same learned content cluster <Cite key="roy2020routing" />. Both choose structural edges before the expensive attention over candidates. Their selection machinery has its own cost and approximation errors; <XRef id="prop-sparse-edge-complexity" /> prices only the retained attention edges after selection.

A second family uses the word *sparse* differently. It computes the dense scores first, then changes their row normalization so the resulting probabilities may be exactly zero.

<Theorem kind="definition" name="Sparsemax normalization" id="def-sparsemax">
For logits $\mathbf z\in\R^q$ and the probability simplex
$$
\Delta^{q-1}\defeq\{\mathbf p\in\R^q:p_i\ge0,\ \textstyle\sum_i p_i=1\},
$$
the **sparsemax** normalization <Cite key="martins2016sparsemax" /> is the Euclidean projection
$$
\operatorname{sparsemax}(\mathbf z)
\defeq
\argmin_{\mathbf p\in\Delta^{q-1}}
\frac12\norm{\mathbf p-\mathbf z}_2^2.
\tag{6}
$$
</Theorem>

<Theorem kind="proposition" name="Sparsemax has a thresholded exact-zero support" id="prop-sparsemax-support">
There is a unique threshold $\tau$ satisfying
$$
\sum_{i=1}^{q}[z_i-\tau]_+=1,
$$
and the projection in (6) is
$$
\operatorname{sparsemax}(\mathbf z)_i=[z_i-\tau]_+.
\tag{7}
$$
Consequently every logit with $z_i\le\tau$ receives probability exactly zero. Sparsemax is also translation-invariant: adding the same constant to every $z_i$ leaves (7) unchanged.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof: attach one multiplier to the simplex sum and one nonnegative multiplier to each constraint $p_i\ge0$. What does complementary slackness say separately on and off the positive support?</MarginNote>

<Theorem kind="proof">
**Existence and uniqueness.** The objective is strictly convex and the simplex is compact and convex, so the minimizer exists and is unique.

**KKT support.** With equality multiplier $\tau$ and inequality multipliers $\nu_i\ge0$, stationarity of
$$
\frac12\sum_i(p_i-z_i)^2+\tau\left(\sum_i p_i-1\right)-\sum_i\nu_i p_i
$$
gives $p_i-z_i+\tau-\nu_i=0$. If $p_i>0$, complementary slackness forces $\nu_i=0$, hence $p_i=z_i-\tau>0$. If $p_i=0$, stationarity gives $\nu_i=\tau-z_i\ge0$, hence $z_i\le\tau$. Combining the cases yields $p_i=[z_i-\tau]_+$.

**Threshold and translation.** The simplex equality determines the unique threshold through $\sum_i[z_i-\tau]_+=1$. Replacing every $z_i$ by $z_i+c$ and $\tau$ by $\tau+c$ preserves every difference and proves translation invariance. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="warning" label="Exact zeros, still dense scores">If sparsemax or $\alpha$-entmax is applied rowwise *after* forming $\mathbf Q\mathbf K^\top$, all $n^2$ scores were already computed. The distribution is sparse, but the attention is not computationally sub-quadratic.</MarginNote>

Adaptively Sparse Transformers learn a per-head $\alpha$ in $\alpha$-entmax, interpolating from softmax at $\alpha=1$ to sparsemax at $\alpha=2$ and assigning exact zeros for suitable logits <Cite key="correia2019adaptively" />. This can change focus, gradients, and interpretability. It becomes a computational saving only if an implementation can identify the support without first scoring every key; exact-zero output alone does not supply that algorithm.

## Natively trainable sparse attention

Modern systems learn the support during pretraining and co-design its granularity with the kernel. Native Sparse Attention (NSA) combines three separately normalized branches: coarse compression for global coverage, selected raw blocks for fine detail, and a recent window for local context <Cite key="yuan2025nsa" />. Formalizing the branch sizes reveals a distinction that the name *sparse* can hide: a small finite density need not be asymptotically sub-quadratic.

<Theorem kind="definition" name="Three-branch Native Sparse Attention support" id="def-nsa-branches">
Let $\ell_c,\delta,r,b,w$ be positive integers. For causal query $\mathbf q_t$, treated as a one-row query matrix, let $c\in\{\mathrm{cmp},\mathrm{slc},\mathrm{win}\}$ index three branch-specific key/value collections $(\widetilde{\mathbf K}_t^c,\widetilde{\mathbf V}_t^c)$. Define an inactive empty branch to return the zero vector:
$$
\mathbf o_t^c
\defeq
\begin{cases}
\operatorname{Attn}(\mathbf q_t,\widetilde{\mathbf K}_t^c,\widetilde{\mathbf V}_t^c), & \lvert\widetilde{\mathbf K}_t^c\rvert>0,\\[2pt]
\mathbf 0, & \lvert\widetilde{\mathbf K}_t^c\rvert=0,
\end{cases}
\qquad
\mathbf o_t
\defeq
\sum_c g_t^c\mathbf o_t^c,
\qquad
g_t^c\in[0,1].
\tag{8}
$$

- **Compression.** Blocks of length $\ell_c$ beginning every $\delta$ positions are mapped to summary key/value rows, giving
  $$
  c_t=\max\!\left(0,1+\left\lfloor\frac{t-\ell_c}{\delta}\right\rfloor\right)
  $$
  compressed rows.
- **Selection.** A score derived from the compressed branch chooses at most $r$ raw-token blocks of width $b$, giving $s_t\le rb$ selected rows.
- **Window.** The most recent $w$ raw rows give $u_t\le\min(t,w)$ local rows.

Because (8) evaluates separate branch attentions, define the per-query and full-sequence support counts with multiplicity:
$$
N_t\defeq c_t+s_t+u_t,
\qquad
M_n\defeq\sum_{t=1}^{n}N_t.
\tag{9}
$$
A source represented in two branches contributes twice to $N_t$.
</Theorem>

<Theorem kind="proposition" name="NSA support savings and the condition for sub-quadratic work" id="prop-nsa-work">
Under <XRef id="def-nsa-branches" />,
$$
M_n
=O\!\left(\frac{n^2}{\delta}+n(rb+w)\right).
\tag{10}
$$
Once the branch collections have been constructed, their one-head score-and-value work is
$$
O\!\left(M_n(d_k+d_v)\right),
\tag{11}
$$
and materialized branch weights occupy $O(M_n)$ storage.

For a given configuration, $N_t\ll t$ is a finite-regime retained-support saving. A sufficient scaling regime for the bound in (10) to be $o(n^2)$ is $\delta\to\infty$ together with $rb+w=o(n)$. Conversely, with fixed $\ell_c,\delta,r,b,w$, the compressed branch alone has $\Theta(n^2/\delta)=\Theta(n^2)$ support, so the configuration is not asymptotically sub-quadratic.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof: the compressed branch keeps roughly one summary per $\delta$ prefix rows. Sum $t/\delta$ over $t=1,\dots,n$ before deciding whether the result is sub-quadratic.</MarginNote>

<Theorem kind="proof">
**Support upper bound.** For every $t$, the compression formula gives $c_t\le1+t/\delta$, while selection and the window give $s_t+u_t\le rb+w$. Therefore
$$
M_n
\le
\sum_{t=1}^{n}\left(1+\frac{t}{\delta}+rb+w\right)
=n(1+rb+w)+\frac{n(n+1)}{2\delta},
$$
which is (10).

**Attention-core cost.** Each of the $M_n$ branch entries contributes one $d_k$-width score dot product and one $d_v$-width weighted value, proving (11) by the same count as <XRef id="prop-sparse-edge-complexity" />.

**Sufficient sub-quadratic regime.** If $\delta\to\infty$, then $n^2/\delta=o(n^2)$; if also $rb+w=o(n)$, then $n(rb+w)=o(n^2)$.

**Fixed-parameter lower bound.** With fixed $\ell_c$ and $\delta$, the compression count satisfies $c_t\ge(t-\ell_c)/\delta$ for all $t\ge\ell_c$, so $\sum_t c_t=\Theta(n^2/\delta)=\Theta(n^2)$. Fixed $r,b,w$ add at most $O(n)$ support and do not change that order. $\qquad\blacksquare$
</Theorem>

For example, the explicit scaling choice $\delta\asymp\sqrt n$ and $rb+w=O(\sqrt n)$ yields $M_n=O(n^{3/2})$. This is a conditional asymptotic example, not the configuration reported by NSA: its finite experiments use fixed branch budgets and establish measured savings, not a new asymptotic class. NSA reports end-to-end training and hardware-aligned kernels, including speedups over its full-attention baseline at $64$K context; those are scoped empirical results of that implementation <Cite key="yuan2025nsa" />. Edge count alone cannot imply wall-clock speed because gather/scatter, branch construction, top-$r$ selection, and memory traffic still enter the roofline equation <XRef id="def-cost-eq" />.

As last verified on **2026-07-13**, three designs illustrate the current progression:

| Design | Learned support | Evidence-scoped distinction |

| **NSA** | gated compression, selected-block, and window branches | trained end-to-end; the paper reports lifecycle speedups for its hardware-aligned implementation <Cite key="yuan2025nsa" /> |
| **MoBA** | each query routes to top-$k$ historical KV blocks, with its causal current block always retained | applies the mixture-of-experts routing analogy to attention blocks and can switch between full and sparse modes <Cite key="lu2025moba" />; compare expert routing in <XRef id="def-moe" /> |
| **DeepSeek Sparse Attention** | a lightweight indexer selects $k$ fine-grained historical tokens per query for the main attention on an MLA-based model | introduced in DeepSeek-V3.2; the main attention is $O(nk)$, while the report's lower-cost lightning indexer remains $O(n^2)$ <Cite key="deepseek2025dsa" />; its reasoning gains are whole-model post-training results, not a causal effect of sparsity |
</Theorem>

This frontier moves quickly, but the accounting does not. Every method must say which pairs are scored, what it costs to choose them, whether the sparse kernel realizes the count, and what approximation boundary replaced the dense map. The next chapter avoids enumerating query-key edges during recurrent execution: a selective state carries history forward instead of revisiting chosen key rows.

Attempt each before expanding its solution.

**Exercise 17.1 (Compute).** For a causal sequence with $n=8$ and sliding-window width $w=3$, list the row degrees, compute $m_{\mathrm{SWA}}$ from both the sum and (3), and compare it with the dense causal edge count. If $d_k=d_v=64$, how many scalar multiply-accumulate coordinates do the score and value stages use under the edge model?

<SolutionBox>

The degrees are $(1,2,3,3,3,3,3,3)$, so $m_{\mathrm{SWA}}=1+2+6\cdot3=21$. Equation (3) agrees: $8\cdot3-3\cdot2/2=21$. Dense causal attention has $8\cdot9/2=36$ edges. Score formation uses $21\cdot64=1{,}344$ coordinate products, and applying values uses another $21\cdot64=1{,}344$, for $2{,}688$ across the two stages, excluding projections and scalar softmax work. $\qquad\blacksquare$

</SolutionBox>

**Exercise 17.2 (Derive).** A depth-$4$ causal stack has $w=5$. Give the exact structural receptive field of row $i=20$ and its size. Then compute the exact clipped edge count on a $14\times14$ image-patch grid whose encoder attention uses a radius-$1$ square neighborhood, and compare it with the uniform per-query upper bound.

<SolutionBox>

Equation (4) gives the earliest row $20-4(5-1)=4$, so $\mathcal R_4(20)=\{4,5,\dots,20\}$ and its size is $17$. Along either grid axis, the two boundary positions have neighborhood width $2$ and the other $12$ positions have width $3$, so the sum of one-dimensional degrees is $2+12\cdot3+2=40$. Square neighborhoods factor across the two axes, giving exactly $40^2=1{,}600$ directed edges. The uniform interior bound is $14^2\cdot3^2=1{,}764$, which is larger because it ignores boundary clipping. The exact attention-core work is $\Theta(1{,}600(d_k+d_v))$ for that fixed grid. $\qquad\blacksquare$

</SolutionBox>

**Exercise 17.3 (Compute).** For the local-plus-strided factorization with $n=4096$, choose $b=64$. Compute the per-row edge upper bound and total edge upper bound. Compare with $n^2$, and prove that $b=\sqrt n$ minimizes the continuous surrogate $b+n/b$.

<SolutionBox>

Here $\lceil n/b\rceil=64$, so each row has at most $64+64=128$ edges and the total is at most $4096\cdot128=524{,}288$. The dense square has $4096^2=16{,}777{,}216$ pairs, exactly $32$ times as many as this upper bound. For $b>0$, AM-GM gives
$$
b+\frac nb\ge2\sqrt{b\cdot n/b}=2\sqrt n,
$$
with equality exactly at $b=n/b$, hence $b=\sqrt n=64$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 17.4 (Compute).** Compute $\operatorname{sparsemax}(1,0.5,-1)$. Then add $7$ to every logit and recompute. Why do the exact zeros not reduce the cost of an attention layer that already formed its dense score matrix?

<SolutionBox>

Assume the first two coordinates are active. Their threshold must satisfy $(1-\tau)+(0.5-\tau)=1$, so $\tau=0.25$. This is consistent because $1,0.5>0.25$ and $-1\le0.25$. Thus sparsemax is $(0.75,0.25,0)$. After adding $7$, the threshold becomes $7.25$ and the differences are unchanged, so the result remains $(0.75,0.25,0)$. The zero arises only during normalization; if all query-key dot products were already formed, the $n^2$ scoring work has already been paid. $\qquad\blacksquare$

</SolutionBox>

**Exercise 17.5 (Analyze).** Why is <XRef id="rem-bigbird-universality" /> neither a corollary of <XRef id="thm-universal-approx" /> nor a result about every causal sparse decoder? Identify three hypotheses that must be carried with the claim.

<SolutionBox>

The dense theorem does not imply that deleting edges preserves its construction; the cited sparse result supplies a different construction through a global-star graph. Its claim fixes the sequence length, measures approximation in $L^p$ for $1<p<\infty$, and assumes positional information plus an auxiliary global row that gathers from and broadcasts to all sequence rows. That graph is noncausal: an early row may obtain information that originated later through the global row. Therefore the result does not automatically apply to a causal decoder or to an arbitrary sparse mask. $\qquad\blacksquare$

</SolutionBox>

**Exercise 17.6 (Compute).** Apply the upper-bound count in <XRef id="prop-nsa-work" /> at $n=65{,}536$, compression stride $\delta=16$, $r=16$ selected blocks of width $b=64$, and window $w=512$. Ignore the additive $n$ ceiling term. Compute the compression and selected-plus-window contributions to $M_n$, compare their sum with dense causal attention, and classify the asymptotic order when these four budgets and the compression-block length $\ell_c$ all remain fixed as $n$ grows.

<SolutionBox>

The compression contribution is
$$
\frac{n(n+1)}{2\delta}
=\frac{65{,}536\cdot65{,}537}{32}
=134{,}219{,}776.
$$
The selected-plus-window budget per query is $rb+w=16\cdot64+512=1{,}536$, contributing $65{,}536\cdot1{,}536=100{,}663{,}296$. The two requested upper-bound terms sum to $234{,}883{,}072$; restoring the omitted ceiling term adds at most $n$. Dense causal attention has $65{,}536\cdot65{,}537/2=2{,}147{,}516{,}416$ edges, about $9.14$ times the two-term approximation. Thus even the restored upper bound proves a large finite-regime reduction, although actual $M_n$ depends on $\ell_c$, the realized $s_t$, and early window clipping. Nevertheless, with fixed $\delta=16$ and fixed $\ell_c$ the compression term is $\Theta(n^2)$; the configuration is sparse by density but not asymptotically sub-quadratic. $\qquad\blacksquare$

</SolutionBox>
