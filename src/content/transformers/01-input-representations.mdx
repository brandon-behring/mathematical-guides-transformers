# Input Representations

**Where we are.** The notation chapter fixed our toolkit — softmax, cross-entropy, and the convex-combination property of row-stochastic matrices. Here we build the *inputs* the transformer consumes. A sequence of discrete tokens must become continuous vectors carrying two things at once: *semantic identity* (which token) and *positional information* (where in the sequence). The second is not optional. Attention, defined in the Attention chapter, is permutation-equivariant — permute the input rows and the output rows permute identically — so without injected position the model cannot tell "dog bites man" from "man bites dog." This chapter constructs identity first as a token lookup, then generalizes it to a single **embedding interface** that carries any modality — text tokens, image patches, discrete codes — into the same space $\R^{n\times d}$. It then builds position by two routes, the *additive* sinusoidal encoding and the *multiplicative* rotary encoding (RoPE), and extends each to the two-dimensional grids that images need, proving the relative-position property throughout.

## Token embeddings

The first step is the simplest: map each token index to a dense vector by a learned lookup, imposing no structure beyond the dimension $d$.

<Theorem kind="definition" name="Vocabulary and token sequence" id="def-tf1-vocabulary">
A **vocabulary** is a finite set $\mathcal V = \{1, 2, \dots, \lvert\mathcal V\rvert\}$ of token indices. A **token sequence** of length $n$ is an ordered tuple $\mathbf x = (x_1, x_2, \dots, x_n) \in \mathcal V^{\,n}$.
</Theorem>

<Theorem kind="definition" name="Token embedding" id="def-tf1-token-embedding">
The **token embedding** is a matrix $\mathbf E \in \R^{\lvert\mathcal V\rvert \times d}$ that maps each token index $x_t \in \mathcal V$ to the dense vector
$$
\mathbf e_t \defeq \mathbf E[x_t, :] \in \R^{d},
$$
the $x_t$-th row of $\mathbf E$. Equivalently, $\mathbf e_t = \mathbf e_{x_t}^\top \mathbf E$, where $\mathbf e_{x_t}$ is the one-hot vector with a $1$ in position $x_t$. The embedded sequence is the matrix $\hat{\mathbf X} = (\mathbf e_1; \mathbf e_2; \dots; \mathbf e_n) \in \R^{n \times d}$, one row per token.
</Theorem>

<MarginNote variant="pattern" label="Lookup = matrix product">A one-hot row times $\mathbf E$ selects a row of $\mathbf E$. The embedding is a linear lookup — no nonlinearity is involved.</MarginNote>

The embedding carries identity but no order: a permutation of the tokens permutes the rows of $\hat{\mathbf X}$ and nothing else. Position must be supplied separately, which is the subject of the later sections. First we generalize the lookup itself.

## From tokens to units: the embedding interface

Token embedding is one instance of a more general operation. A modality reaches the transformer through a single map from its own raw units into $\R^{d}$; once that map is fixed, the model sees only the matrix it produces. We name the interface, then read token lookup, image patches, and discrete visual codes off it as three instances of one form.

<Theorem kind="definition" name="Embedding interface" id="def-tf1-embedding-interface">
A **modality** is a pair $(\mathcal U, \Emb)$ of a **unit space** $\mathcal U$ and an **embedding map** $\Emb:\mathcal U\to\R^{d}$. Given a raw input parsed into an ordered tuple of units $(u_1,\dots,u_n)\in\mathcal U^{\,n}$, the **embedded sequence** is
$$
\hat{\mathbf X} \defeq \big(\Emb(u_1);\,\Emb(u_2);\,\dots;\,\Emb(u_n)\big)\in\R^{n\times d}.
$$
The transformer core acts on $\hat{\mathbf X}$ alone; the modality is confined to the choice of $(\mathcal U,\Emb)$. Token embedding (<XRef id="def-tf1-token-embedding" />) is the instance $\mathcal U=\mathcal V$, $\Emb(x)=\mathbf E[x,:]=\mathbf e_x^\top\mathbf E$.
</Theorem>

<MarginNote variant="pattern" label="One linear form">Token lookup is patch projection on a one-hot input: $\mathbf e_x^\top\mathbf E$ versus $\operatorname{vec}(\mathbf u)^\top\mathbf W_{\text{patch}}$ — the same map, a dense input in place of a one-hot.</MarginNote>

<Theorem kind="definition" name="Patch embedding (ViT front-end)" id="def-tf1-patch-embedding">
An image $\mathbf I\in\R^{H\times W\times c}$ with $p\mid H$ and $p\mid W$ is partitioned into a grid of $H_p\times W_p$ non-overlapping $p\times p$ patches, $H_p=H/p$, $W_p=W/p$, giving $n=H_pW_p$ units. The unit space is $\mathcal U=\R^{p\times p\times c}$ and the embedding is a linear projection of the flattened patch,
$$
\Emb(\mathbf u_{ab}) \defeq \operatorname{vec}(\mathbf u_{ab})^\top\,\mathbf W_{\text{patch}},
\qquad \mathbf W_{\text{patch}}\in\R^{p^2c\times d},
$$
for grid coordinate $(a,b)$; the $n$ patch vectors, in raster order, form $\hat{\mathbf X}\in\R^{n\times d}$.<Sidenote>The Vision Transformer front-end <Cite key="dosovitskiy2021vit" />.</Sidenote>
</Theorem>

<Theorem kind="definition" name="Codebook (discrete visual) embedding" id="def-tf1-codebook-embedding">
A **codebook** is a matrix $\mathbf C\in\R^{K\times d}$ of $K$ learned codes. With unit space $\mathcal U=\{1,\dots,K\}$, the embedding is the row lookup
$$
\Emb(z) \defeq \mathbf C[z,:] = \mathbf e_z^\top\mathbf C,
$$
**algebraically identical** to the token lookup of <XRef id="def-tf1-token-embedding" />, with $\mathbf C$ in the role of $\mathbf E$. Image tokens produced by a learned discrete autoencoder (a later chapter) are embedded by exactly the text pathway.<Sidenote>Discrete visual tokens via vector quantization <Cite key="vandenoord2017vqvae" />.</Sidenote>
</Theorem>

The three differ only in $\mathcal U$ and how $\Emb$ is realized; the patch case looks new only because its input is a dense vector rather than a one-hot index. The next result removes even that appearance of novelty, for the instance that looks least like a lookup.

<Theorem kind="theorem" name="Patch embedding is a strided convolution" id="thm-tf1-patch-conv">
Let $\mathbf W_{\text{patch}}\in\R^{p^2c\times d}$ define the patch embedding of <XRef id="def-tf1-patch-embedding" />, and build a kernel $\mathbf K\in\R^{p\times p\times c\times d}$ by reshaping each column $\mathbf W_{\text{patch}}[:,j]$ into the filter $\mathbf K[:,:,:,j]$ with the same index order as $\operatorname{vec}$. Then for every image partitioned into non-overlapping patches $\{\mathbf u_{ab}\}$,
$$
\big[\,\mathbf I \star_p \mathbf K\,\big]_{a,b,j} = \Emb(\mathbf u_{ab})_j,
$$
where $\star_p$ is the stride-$p$, kernel-size-$p$ cross-correlation with no padding — a `Conv2d(c, d, kernel_size=p, stride=p, bias=False)` whose weight is $\mathbf K$ in PyTorch's $(d,c,p,p)$ layout. The patch-embedding layer and that convolution compute the same map; they differ only in how the identical parameters are indexed (and the convolution must carry no bias).
</Theorem>

<Theorem kind="proof">
Two facts assemble the claim: the windows tile the image, and one window is one inner product.

**The windows are the patches.** Because the stride equals the kernel size, the convolution windows are disjoint and cover the image; window $(a,b)$ is exactly the patch $\mathbf u_{ab}$.

**One output channel is one Frobenius product.** By the definition of cross-correlation, output channel $j$ at window $(a,b)$ is
$$
\langle \mathbf K[:,:,:,j],\,\mathbf u_{ab}\rangle_F = \sum_{r,s,e}\mathbf K[r,s,e,j]\,\mathbf u_{ab}[r,s,e].
$$
By construction $\mathbf K[:,:,:,j]$ is the reshape of column $j$ of $\mathbf W_{\text{patch}}$, and the reshape and $\operatorname{vec}$ share an index order, so this sum equals $\operatorname{vec}(\mathbf u_{ab})^\top\mathbf W_{\text{patch}}[:,j] = \Emb(\mathbf u_{ab})_j$. The windows tile the image, so the maps agree on every patch. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="tip" label="Why it's not new">Patchification is the degenerate convolution where stride equals kernel: the windows do not overlap, so the one shared linear map $\mathbf W_{\text{patch}}$ is applied to each patch independently, with no cross-patch mixing.</MarginNote>

## Sinusoidal positional encoding

The concept image is a clock. Picture each pair of coordinates as the tip of a hand sweeping a circle at its own fixed rate; the position is the angle, and advancing one step turns every hand by a fixed amount. We add such a vector of angles to the token embedding. The sinusoidal encoding of <Cite key="vaswani2017attention" /> realizes this with no learned parameters and two properties we will prove: a guaranteed collision-free window for nearby positions, and a *relative*-position structure in which a shift is a fixed rotation.

<MarginNote variant="formula" label="Sinusoidal PE">$\operatorname{PE}(t,2i)=\sin(\omega_i t)$, $\operatorname{PE}(t,2i{+}1)=\cos(\omega_i t)$, with $\omega_i = 10000^{-2i/d}$.</MarginNote>

<Theorem kind="definition" name="Sinusoidal positional encoding" id="def-tf1-sinusoidal-pe">
For position $t \in \{0,1,2,\dots\}$ and even dimension $d$, define $\operatorname{PE} : \Z_{\ge 0} \to \R^{d}$ by
$$
\operatorname{PE}(t, 2i) \defeq \sin(\omega_i\, t),
\qquad
\operatorname{PE}(t, 2i+1) \defeq \cos(\omega_i\, t),
$$
for $i = 0, 1, \dots, d/2 - 1$, where the **angular frequency** of pair $i$ is
$$
\omega_i \defeq \frac{1}{10000^{2i/d}}.
$$
The frequencies decrease geometrically from $\omega_0 = 1$ (fastest) to $\omega_{d/2-1} = 10000^{-1 + 2/d}$ (slowest, near $1/10000$ for large $d$).
</Theorem>

<Theorem kind="definition" name="Transformer input" id="def-tf1-input">
The input to the first transformer layer is the entrywise sum of the token embedding and the positional encoding:
$$
\mathbf X \defeq \hat{\mathbf X} + \mathbf P,
\qquad P_{t, j} = \operatorname{PE}(t, j),
$$
where $\hat{\mathbf X} \in \R^{n\times d}$ is the embedded sequence (<XRef id="def-tf1-token-embedding" />) and $\mathbf P \in \R^{n\times d}$ is the positional encoding matrix. The token embedding supplies lexical identity; $\mathbf P$ supplies order, in the same $d$-dimensional space.
</Theorem>

We now prove the two properties. The first — that a positional shift acts as a fixed linear map — is the structural heart of the construction.

<Theorem kind="theorem" name="Relative position is a fixed rotation" id="thm-tf1-pe-rotation">
For any fixed offset $k \in \Z$ there is a block-diagonal matrix $\mathbf M_k \in \R^{d\times d}$, depending on $k$ but not on $t$, such that
$$
\operatorname{PE}(t + k) = \operatorname{PE}(t)\,\mathbf M_k^\top \qquad \text{for all } t.
$$
Here $\mathbf M_k = \operatorname{diag}\!\big(\mathbf R_0(k), \mathbf R_1(k), \dots, \mathbf R_{d/2-1}(k)\big)$, and each $2\times 2$ block is the rotation
$$
\mathbf R_i(k) =
\begin{pmatrix}
\cos(\omega_i k) & \sin(\omega_i k) \\
-\sin(\omega_i k) & \cos(\omega_i k)
\end{pmatrix}.
$$
</Theorem>

<Theorem kind="proof">
We show the claim on a single pair of dimensions; independence across pairs then assembles the block-diagonal map.

**Reduce to one pair.** By <XRef id="def-tf1-sinusoidal-pe" />, the $i$-th pair of $\operatorname{PE}(t+k)$ is $\big(\sin(\omega_i(t+k)),\, \cos(\omega_i(t+k))\big)$.

**Apply the angle-addition identities.** Expanding $\sin$ and $\cos$ of $\omega_i t + \omega_i k$,
$$
\begin{aligned}
\sin(\omega_i t + \omega_i k) &= \cos(\omega_i k)\sin(\omega_i t) + \sin(\omega_i k)\cos(\omega_i t), \\
\cos(\omega_i t + \omega_i k) &= -\sin(\omega_i k)\sin(\omega_i t) + \cos(\omega_i k)\cos(\omega_i t).
\end{aligned}
$$

**Read it as a matrix.** With the pair written as a column $\big(\operatorname{PE}(t,2i),\, \operatorname{PE}(t,2i{+}1)\big)^\top$, the two lines say
$$
\begin{pmatrix} \operatorname{PE}(t+k, 2i) \\ \operatorname{PE}(t+k, 2i+1) \end{pmatrix}
= \mathbf R_i(k)
\begin{pmatrix} \operatorname{PE}(t, 2i) \\ \operatorname{PE}(t, 2i+1) \end{pmatrix}.
$$

**Assemble.** Each pair transforms independently by its own $\mathbf R_i(k)$, so the full vector transforms by the block-diagonal $\mathbf M_k$; transposing converts the column-vector action into the stated row-vector form $\operatorname{PE}(t+k) = \operatorname{PE}(t)\,\mathbf M_k^\top$. As $\mathbf R_i(k)$ involves only $\omega_i k$, the map is independent of $t$. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="pattern" label="Relative, not absolute">$\mathbf M_k$ depends only on the offset $k$, not on $t$ — which is exactly what makes sinusoidal PE encode *relative* position.</MarginNote>

A consequence used by attention: the inner product $\operatorname{PE}(t)\cdot\operatorname{PE}(t+k)$ depends only on $k$, not on $t$. Each pair contributes $\sin(\omega_i t)\sin(\omega_i(t+k)) + \cos(\omega_i t)\cos(\omega_i(t+k)) = \cos(\omega_i k)$ by the same addition identity, so the model can read relative position off a linear function of the encodings, with no explicit subtraction of positions.

<Theorem kind="theorem" name="Collision-free window" id="thm-tf1-pe-collision">
For any two distinct positions $t \ne t'$ with
$$
\lvert t - t'\rvert < 2\pi \cdot 10000^{\,1 - 2/d},
$$
the sinusoidal encoding is injective: $\operatorname{PE}(t) \ne \operatorname{PE}(t')$.
</Theorem>

<Theorem kind="proof">
We argue by contraposition, using only the *slowest* pair.

**Suppose a collision.** If $\operatorname{PE}(t) = \operatorname{PE}(t')$, every pair agrees; in particular the slowest pair $i = d/2 - 1$ gives $\sin(\omega_{d/2-1} t) = \sin(\omega_{d/2-1} t')$ and $\cos(\omega_{d/2-1} t) = \cos(\omega_{d/2-1} t')$.

**Equal sine and cosine forces equal angle mod $2\pi$.** A point on the unit circle is determined by its sine and cosine together, so $\omega_{d/2-1}(t - t') \in 2\pi\Z$.

**Bound the gap.** With $t \ne t'$ the smallest nonzero value of $\lvert\omega_{d/2-1}(t-t')\rvert$ is $2\pi$, hence
$$
\lvert t - t'\rvert \ge \frac{2\pi}{\omega_{d/2-1}} = 2\pi \cdot 10000^{\,1 - 2/d}.
$$
So any gap shorter than this cannot produce a collision; equivalently, $\operatorname{PE}$ is injective on every window narrower than $2\pi \cdot 10000^{1-2/d}$. $\qquad\blacksquare$
</Theorem>

This is a *guaranteed* lower bound from one pair, not the exact period of the full vector. A genuine full-vector collision needs every frequency pair to realign simultaneously, which generically happens far later; the bound is the conservative window inside which uniqueness is certain. Beyond it, the slowest coordinates begin to wrap, and modern models prefer the rotary scheme below or relative-bias schemes for long-range structure.

## Two-dimensional positions

A one-dimensional sequence index generalizes to a grid coordinate $(a,b)$ with no new abstraction. Images need it: the patch embedding of <XRef id="def-tf1-patch-embedding" /> lays patches out in raster order, but a patch's neighbors above and below are as relevant as those left and right. We spend half the channels on the row and half on the column, and the relative-position law of <XRef id="thm-tf1-pe-rotation" /> holds on each half.

<Theorem kind="definition" name="Axial (factorized) 2D sinusoidal PE" id="def-tf1-pe-2d">
For a patch at grid coordinate $(a,b)$ and $d$ divisible by $4$, split the channels into a row half and a column half, each carrying the 1D sinusoidal encoding of <XRef id="def-tf1-sinusoidal-pe" /> on $d/2$ channels:
$$
\operatorname{PE}_{\text{2D}}(a,b) \defeq \big(\,\operatorname{PE}(a),\ \operatorname{PE}(b)\,\big)\in\R^{d},
$$
where each half uses the frequencies $\omega_i = 10000^{-2i/(d/2)}$ for $i=0,\dots,d/4-1$.
</Theorem>

<MarginNote variant="pattern" label="Reuse, not re-derive">2D position is two copies of <XRef id="thm-tf1-pe-rotation" /> on orthogonal channel blocks — no new machinery.</MarginNote>

<Theorem kind="theorem" name="Factorized 2D shift is a fixed block rotation" id="thm-tf1-pe-2d-shift">
For any grid offset $(k,l)\in\Z^2$ there is a block-diagonal orthogonal matrix $\mathbf M_{k,l}=\operatorname{diag}\!\big(\mathbf M^{\text{row}}_k,\,\mathbf M^{\text{col}}_l\big)\in\R^{d\times d}$, independent of $(a,b)$, with
$$
\operatorname{PE}_{\text{2D}}(a+k,\,b+l) = \operatorname{PE}_{\text{2D}}(a,b)\,\mathbf M_{k,l}^\top.
$$
Consequently $\operatorname{PE}_{\text{2D}}(a,b)\cdot\operatorname{PE}_{\text{2D}}(a+k,b+l)$ depends only on the offset $(k,l)$, with closed form
$$
\sum_{i=0}^{d/4-1}\cos(\omega_i k) \;+\; \sum_{i=0}^{d/4-1}\cos(\omega_i l).
$$
</Theorem>

<Theorem kind="proof">
Concatenation makes the two halves act independently. Applying <XRef id="thm-tf1-pe-rotation" /> to the row half gives $\operatorname{PE}(a+k)=\operatorname{PE}(a)\,(\mathbf M^{\text{row}}_k)^\top$ and to the column half $\operatorname{PE}(b+l)=\operatorname{PE}(b)\,(\mathbf M^{\text{col}}_l)^\top$, each $\mathbf M$ being the block-diagonal rotation of its offset. Stacking the two independent blocks yields the block-diagonal $\mathbf M_{k,l}$, which depends only on $(k,l)$. For the inner product, the corollary of <XRef id="thm-tf1-pe-rotation" /> makes each frequency pair contribute $\cos(\omega_i\cdot\text{offset})$; summing over the row half and the column half gives the stated closed form. $\qquad\blacksquare$
</Theorem>

## Rotary position embedding (RoPE)

Sinusoidal PE *adds* position to the input. RoPE instead *rotates* the query and key vectors inside attention, so that the query-key dot product — the only place position needs to act — depends exactly on the relative offset. The concept image is again rotating hands, but now the data vectors themselves are the hands: rotate the query by an angle set by its position and the key by an angle set by its, and only the difference of angles survives the inner product. We first fix the rotation we use.

<Theorem kind="definition" name="2D rotation matrix" id="def-tf1-rotation-2d">
For angle $\theta \in \R$, the $2\times 2$ rotation matrix is
$$
\mathbf R(\theta) \defeq
\begin{pmatrix}
\cos\theta & -\sin\theta \\
\sin\theta & \cos\theta
\end{pmatrix}.
$$
It is orthogonal, with $\mathbf R(\theta)^\top = \mathbf R(-\theta) = \mathbf R(\theta)^{-1}$, and rotations compose additively: $\mathbf R(\alpha)\mathbf R(\beta) = \mathbf R(\alpha + \beta)$.
</Theorem>

<Theorem kind="definition" name="Rotary position embedding" id="def-tf1-rope">
Given a query $\mathbf q \in \R^{d}$ at position $m$ and a key $\mathbf k \in \R^{d}$ at position $t$, RoPE applies a position-dependent rotation to each $2$D subspace before the dot product:
$$
f(\mathbf q, m) \defeq \mathbf\Theta(m)\,\mathbf q,
\qquad
f(\mathbf k, t) \defeq \mathbf\Theta(t)\,\mathbf k,
$$
where
$$
\mathbf\Theta(t) \defeq \operatorname{diag}\!\big(\mathbf R(\theta_0 t),\, \mathbf R(\theta_1 t),\, \dots,\, \mathbf R(\theta_{d/2-1} t)\big)
$$
is block-diagonal in the $2\times 2$ rotations of <XRef id="def-tf1-rotation-2d" />, and the base angles $\theta_i = 10000^{-2i/d}$ follow the same geometric schedule as the sinusoidal frequencies (<XRef id="def-tf1-sinusoidal-pe" />). Here $\mathbf q, \mathbf k$ are treated as **column** vectors — as in the 2D-shift proof — and transposing converts the rotation action to the row form.
</Theorem>

<MarginNote variant="pattern" label="Position in the dot product">RoPE writes relative position directly into the query-key inner product — no fixed vector is added and no positions are subtracted.</MarginNote>

<MarginNote variant="tip" label="Self-explanation">Before reading the proof: $\mathbf\Theta(t)$ is built from orthogonal rotations that compose additively. What does $\mathbf\Theta(m)^\top\mathbf\Theta(t)$ have to be?</MarginNote>

<Theorem kind="theorem" name="RoPE relative-position property" id="thm-tf1-rope-relative">
The dot product of RoPE-encoded query and key depends only on the relative position $t - m$:
$$
f(\mathbf q, m)^\top f(\mathbf k, t) = \mathbf q^\top\, \mathbf\Theta(t - m)\, \mathbf k.
$$
</Theorem>

<Theorem kind="proof">
The high-level step is to collapse the two position rotations into one by orthogonality; the rest is bookkeeping over blocks.

**Expand the dot product.** By <XRef id="def-tf1-rope" />,
$$
f(\mathbf q, m)^\top f(\mathbf k, t)
= (\mathbf\Theta(m)\mathbf q)^\top (\mathbf\Theta(t)\mathbf k)
= \mathbf q^\top \mathbf\Theta(m)^\top \mathbf\Theta(t)\, \mathbf k.
$$

**Transpose is negative rotation (blockwise).** Each diagonal block is orthogonal with $\mathbf R(\theta_i m)^\top = \mathbf R(-\theta_i m)$ (<XRef id="def-tf1-rotation-2d" />), so $\mathbf\Theta(m)^\top = \mathbf\Theta(-m)$.

**Compose additively (blockwise).** For each block, $\mathbf R(-\theta_i m)\mathbf R(\theta_i t) = \mathbf R(\theta_i(t - m))$ by the composition rule of <XRef id="def-tf1-rotation-2d" />; assembling the blocks gives $\mathbf\Theta(-m)\mathbf\Theta(t) = \mathbf\Theta(t - m)$.

**Substitute.** Therefore $f(\mathbf q, m)^\top f(\mathbf k, t) = \mathbf q^\top \mathbf\Theta(t-m)\mathbf k$, a function of $t - m$ alone. $\qquad\blacksquare$
</Theorem>

<Theorem kind="remark" name="Complex-number viewpoint" id="rem-tf1-rope-complex">
RoPE is cleanest in $\mathbb C^{\,d/2}$. Pair adjacent coordinates into $z_i = x_{2i} + \mathrm i\, x_{2i+1}$; the rotation $\mathbf R(\theta_i t)$ becomes multiplication by $e^{\mathrm i\theta_i t}$, so $f(\mathbf q, m)_i = q_i\, e^{\mathrm i\theta_i m}$ and $f(\mathbf k, t)_i = k_i\, e^{\mathrm i\theta_i t}$. The (Hermitian) dot product is then $\sum_i \overline{q_i}\,k_i\, e^{\mathrm i\theta_i(t-m)}$, manifestly a function of $t - m$. This is also how RoPE is implemented: an entrywise complex multiply costs $O(d)$, not the $O(d^2)$ of a dense matrix product.
</Theorem>

<Theorem kind="remark" name="Sinusoidal PE versus RoPE" id="rem-tf1-pe-vs-rope">
Both schemes use the geometric schedule $\theta_i = 10000^{-2i/d}$, but they inject position differently. Sinusoidal PE *adds* a fixed vector to the input (<XRef id="def-tf1-input" />), so its relative-position structure (<XRef id="thm-tf1-pe-rotation" />) survives only approximately once the learned projections $\mathbf W^Q, \mathbf W^K$ act. RoPE *multiplies* inside attention (<XRef id="def-tf1-rope" />), so the relative-position identity (<XRef id="thm-tf1-rope-relative" />) holds *exactly* in the dot product. Open decoder-only models such as LLaMA adopt RoPE for this reason.<Sidenote>RoPE was introduced in 2021 by <Cite key="su2024roformer" /> (journal version 2024); its use in a large open model is documented in <Cite key="touvron2023llama" />.</Sidenote>
</Theorem>

<Theorem kind="remark" name="Axial RoPE" id="rem-tf1-rope-2d">
RoPE extends to a grid the same way the sinusoidal scheme did in <XRef id="thm-tf1-pe-2d-shift" />: assign half the rotary pairs the row angle $\theta_i a$ and half the column angle $\theta_i b$. <XRef id="thm-tf1-rope-relative" /> then holds per axis, so the rotated query-key dot product depends only on the 2D offset $(a-a',\,b-b')$. This is the position scheme of modern vision and multimodal transformers over patch grids.
</Theorem>

<Theorem kind="remark" name="The core cannot tell which modality produced X" id="rem-tf1-modality-blind">
Once $\mathbf X = \hat{\mathbf X} + \mathbf P$ is formed, it is a point in $\R^{n\times d}$ carrying no record of whether it came from token lookup (<XRef id="def-tf1-token-embedding" />), patch projection (<XRef id="def-tf1-patch-embedding" />), or codebook lookup (<XRef id="def-tf1-codebook-embedding" />), nor whether position was injected over a 1D index or a 2D grid. Every result in the chapters that follow — attention and its variance scaling, multi-head factorization, the feed-forward and layer-norm sublayers, residual stacking, the training loss, and the inference optimizations — is stated over $\R^{n\times d}$ and never references a vocabulary, so each transfers verbatim. Modality is **fully discharged at this boundary**; the transformer core is modality-blind. We number the three interfaces where modality may enter, for later chapters to cite: **Interface (1)** the embedding boundary $\Emb$; **Interface (2)** the positional structure $\operatorname{PE}$; **Interface (3)** the read-out and cross-modal coupling (output heads and cross-attention). Everything between them is the shared core. A learned modality-type vector or a $\texttt{[CLS]}$ slot, added to $\hat{\mathbf X}$ exactly as position is, is the only further interface-level marking a multimodal model needs.
</Theorem>

## Other positional schemes

Three further schemes recur in practice and contrast usefully with the two above. A **learned positional encoding** is a parameter matrix $\mathbf P \in \R^{n_{\max}\times d}$ trained jointly with the model, giving position $t$ the row $\mathbf P[t, :]$; it is flexible at a fixed context length but, having no closed form, cannot extrapolate beyond $n_{\max}$. **ALiBi** leaves the query and key vectors untouched and instead subtracts a head-specific linear penalty $m_r\,(i - j)$ from each logit before softmax, encoding an explicit monotone preference for nearby tokens that is defined for any distance and so extrapolates naturally<Cite key="press2022alibi" />. RoPE and ALiBi both encode *relative* position but in different places — RoPE in the geometry of the query-key vectors (<XRef id="thm-tf1-rope-relative" />), ALiBi in the logits after the dot product. Finally, for patch grids a **learned 2D positional encoding** factorizes the table into row and column halves, $\mathbf P^{\text{row}}_a + \mathbf P^{\text{col}}_b$ — the learned counterpart of the axial construction of <XRef id="def-tf1-pe-2d" />.

We summarize the five schemes by where they inject position and how they extrapolate.

| Method | Relative-position mechanism | Extrapolation |
|---|---|---|
| Sinusoidal PE | Additive input features; an offset becomes a fixed rotation (<XRef id="thm-tf1-pe-rotation" />), exact only before the $\mathbf W^Q,\mathbf W^K$ projections | Defined for all positions; periodicity weakens very-long-range uniqueness |
| RoPE | Multiplicative rotations applied directly in the query/key subspaces (<XRef id="thm-tf1-rope-relative" />) | Relative effect is exact in the dot product; strong long-context behavior |
| Learned PE | Absolute-position lookup $\mathbf P[t, :]$ | None beyond $n_{\max}$ |
| ALiBi | Head-specific linear distance bias added to the logits | Natural — the bias is defined for arbitrary distances |
| Learned 2D PE | Separable row/column tables $\mathbf P^{\text{row}} + \mathbf P^{\text{col}}$ over the patch grid | None beyond the trained grid; interpolated for new resolutions |

The Vision Transformer originally used a *learned 1D* table over the raster patch index, observing **no significant gain** from explicitly 2D-aware encodings — in the reported ablation the 2D variant scored marginally *worse* — since self-attention recovers grid structure from the data.<Sidenote>The ablation is reported in the original ViT study <Cite key="dosovitskiy2021vit" />.</Sidenote>

## Exercises

Attempt each before expanding its solution.

**Exercise 1.1 (Compute).** Let $d = 4$, so $\omega_0 = 1$ and $\omega_1 = 10000^{-1/2} = 0.01$. Write out $\operatorname{PE}(0)$ and $\operatorname{PE}(1)$ numerically, and confirm $\operatorname{PE}(0) \ne \operatorname{PE}(1)$.

<SolutionBox>

With $d = 4$ the frequencies are $\omega_0 = 10000^{-0/4} = 1$ and $\omega_1 = 10000^{-2/4} = 0.01$. By <XRef id="def-tf1-sinusoidal-pe" />, $\operatorname{PE}(t) = \big(\sin(\omega_0 t),\, \cos(\omega_0 t),\, \sin(\omega_1 t),\, \cos(\omega_1 t)\big)$, so
$$
\operatorname{PE}(0) = (0,\, 1,\, 0,\, 1), \qquad
\operatorname{PE}(1) \approx (0.8415,\, 0.5403,\, 0.0100,\, 1.0000).
$$
They differ already in the fast pair ($\sin 1 \approx 0.8415 \ne 0$), so $\operatorname{PE}(0) \ne \operatorname{PE}(1)$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.2 (Compute).** Continuing with $d = 4$, form $\mathbf R_0(1)$ and apply it to the first pair of $\operatorname{PE}(0)$; check that the result equals the first pair of $\operatorname{PE}(1)$, verifying <XRef id="thm-tf1-pe-rotation" /> on one block.

<SolutionBox>

For offset $k = 1$ and pair $i = 0$ ($\omega_0 = 1$), the rotation is $\mathbf R_0(1) = \begin{pmatrix} \cos 1 & \sin 1 \\ -\sin 1 & \cos 1 \end{pmatrix}$. The first pair of $\operatorname{PE}(0)$ is $(0, 1)^\top$, and
$$
\mathbf R_0(1)\begin{pmatrix} 0 \\ 1 \end{pmatrix} = \begin{pmatrix} \sin 1 \\ \cos 1 \end{pmatrix} \approx \begin{pmatrix} 0.8415 \\ 0.5403 \end{pmatrix},
$$
exactly the first pair of $\operatorname{PE}(1)$. This verifies $\operatorname{PE}(t+k) = \mathbf R_i(k)\,\operatorname{PE}(t)$ on the block. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.3 (Prove).** Show that the inner product $\operatorname{PE}(t)\cdot\operatorname{PE}(t+k)$ depends only on the offset $k$, not on $t$, and give its closed form as a sum over frequency pairs.

<SolutionBox>

For pair $i$, the contribution to the inner product is
$$
\sin(\omega_i t)\sin(\omega_i(t+k)) + \cos(\omega_i t)\cos(\omega_i(t+k)) = \cos(\omega_i k),
$$
using $\cos(A - B) = \cos A\cos B + \sin A\sin B$. Summing over the $d/2$ pairs, $\operatorname{PE}(t)\cdot\operatorname{PE}(t+k) = \sum_{i=0}^{d/2-1}\cos(\omega_i k)$, which depends only on the offset $k$, not on $t$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.4 (Derive).** A vocabulary lookup table $\mathbf E$ admits no analogue of <XRef id="thm-tf1-pe-rotation" />: prove there is in general no fixed map sending the embedding of token $x$ to that of token $x+1$ for every $x$, and explain in one sentence what structural feature of $\operatorname{PE}$ makes the shift law possible.

<SolutionBox>

Suppose a fixed map sent each token's embedding to the next, $\mathbf E[x+1,:] = g(\mathbf E[x,:])$ for one fixed $g$. The rows of $\mathbf E$ are free parameters, so choose $x, x'$ with $\mathbf E[x,:] = \mathbf E[x',:]$ yet $\mathbf E[x+1,:] \ne \mathbf E[x'+1,:]$; then $g$ must send one input to two outputs — a contradiction. The index $x{+}1$ is an arbitrary symbol, not a geometric increment. What makes $\operatorname{PE}$'s shift law possible is that position enters through angles $\omega_i t$, where adding $k$ is the same rotation $\mathbf R_i(k)$ for every $t$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.5 (Prove).** Using only <XRef id="def-tf1-rotation-2d" />, establish $\mathbf\Theta(s)\mathbf\Theta(t) = \mathbf\Theta(s+t)$ and $\mathbf\Theta(t)^\top = \mathbf\Theta(-t)$ for the block-diagonal RoPE map, and deduce <XRef id="thm-tf1-rope-relative" /> in one line.

<SolutionBox>

Each diagonal block of $\mathbf\Theta(\cdot)$ is $\mathbf R(\theta_i\,\cdot)$. By <XRef id="def-tf1-rotation-2d" />, $\mathbf R(\theta_i s)\mathbf R(\theta_i t) = \mathbf R(\theta_i(s+t))$ and $\mathbf R(\theta_i t)^\top = \mathbf R(-\theta_i t)$; blockwise this gives $\mathbf\Theta(s)\mathbf\Theta(t) = \mathbf\Theta(s+t)$ and $\mathbf\Theta(t)^\top = \mathbf\Theta(-t)$. Then $f(\mathbf q,m)^\top f(\mathbf k,t) = \mathbf q^\top\mathbf\Theta(m)^\top\mathbf\Theta(t)\mathbf k = \mathbf q^\top\mathbf\Theta(t-m)\mathbf k$, which is <XRef id="thm-tf1-rope-relative" />. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.6 (Prove).** Exhibit a weight matrix $\mathbf W_{\text{patch}}$ and inputs making patch embedding (<XRef id="def-tf1-patch-embedding" />) reduce to token embedding (<XRef id="def-tf1-token-embedding" />), and conclude that token lookup is the one-hot special case of patch projection.

<SolutionBox>

Set the flattened-patch dimension to the vocabulary size, $p^2 c = \lvert\mathcal V\rvert$, and take $\mathbf W_{\text{patch}} = \mathbf E\in\R^{\lvert\mathcal V\rvert\times d}$. For token $x$, feed the patch whose flattening is the one-hot $\mathbf e_x$ (a "pixel" array that is $1$ at flat index $x$ and $0$ elsewhere). Then, by <XRef id="def-tf1-patch-embedding" />,
$$
\Emb(\mathbf u) = \operatorname{vec}(\mathbf u)^\top\mathbf W_{\text{patch}} = \mathbf e_x^\top\mathbf E = \mathbf E[x,:],
$$
the token embedding of <XRef id="def-tf1-token-embedding" />. Token lookup is thus patch projection restricted to one-hot inputs; a general image patch is a *dense* input to the very same linear map $\operatorname{vec}(\cdot)^\top\mathbf W$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.7 (Compute).** For a $4\times 4\times 1$ image with $p = 2$, list the $n = 4$ patches, give the shapes of $\mathbf W_{\text{patch}}$ and $\hat{\mathbf X}$ for $d = 3$, build the kernel $\mathbf K$ of <XRef id="thm-tf1-patch-conv" />, and verify that one output channel equals the corresponding patch Frobenius product.

<SolutionBox>

With $H = W = 4$, $c = 1$, $p = 2$ we get $H_p = W_p = 2$, so $n = 4$ patches, each $2\times 2\times 1$ flattening to dimension $p^2 c = 4$. Writing the image entries $I_{rs}$, the four patches are the disjoint $2\times2$ blocks
$$
\mathbf u_{00} = \left(\begin{smallmatrix} I_{00} & I_{01}\\ I_{10} & I_{11}\end{smallmatrix}\right),\
\mathbf u_{01} = \left(\begin{smallmatrix} I_{02} & I_{03}\\ I_{12} & I_{13}\end{smallmatrix}\right),\
\mathbf u_{10} = \left(\begin{smallmatrix} I_{20} & I_{21}\\ I_{30} & I_{31}\end{smallmatrix}\right),\
\mathbf u_{11} = \left(\begin{smallmatrix} I_{22} & I_{23}\\ I_{32} & I_{33}\end{smallmatrix}\right).
$$
For $d = 3$, $\mathbf W_{\text{patch}}\in\R^{4\times 3}$ and $\hat{\mathbf X}\in\R^{4\times 3}$. The kernel reshapes each column $\mathbf W_{\text{patch}}[:,j] = (w_{1j},w_{2j},w_{3j},w_{4j})$ in row-major order to $\mathbf K[:,:,:,j] = \left(\begin{smallmatrix} w_{1j} & w_{2j}\\ w_{3j} & w_{4j}\end{smallmatrix}\right)$. Then, for patch $\mathbf u_{00}$ with $\operatorname{vec}(\mathbf u_{00}) = (I_{00},I_{01},I_{10},I_{11})$,
$$
\langle \mathbf K[:,:,:,j], \mathbf u_{00}\rangle_F = w_{1j}I_{00} + w_{2j}I_{01} + w_{3j}I_{10} + w_{4j}I_{11} = \operatorname{vec}(\mathbf u_{00})^\top\mathbf W_{\text{patch}}[:,j] = \Emb(\mathbf u_{00})_j,
$$
matching <XRef id="thm-tf1-patch-conv" />. $\qquad\blacksquare$

</SolutionBox>

**Exercise 1.8 (Prove).** Deduce the factorized 2D shift law (<XRef id="thm-tf1-pe-2d-shift" />) from the 1D shift law (<XRef id="thm-tf1-pe-rotation" />) by applying it to each channel half, and write out the closed form of the 2D inner product.

<SolutionBox>

Write $\operatorname{PE}_{\text{2D}}(a,b) = (\operatorname{PE}(a),\operatorname{PE}(b))$ on disjoint row and column halves. For offset $(k,l)$,
$$
\operatorname{PE}_{\text{2D}}(a+k,b+l) = (\operatorname{PE}(a+k),\,\operatorname{PE}(b+l)).
$$
<XRef id="thm-tf1-pe-rotation" /> gives $\operatorname{PE}(a+k) = \operatorname{PE}(a)(\mathbf M^{\text{row}}_k)^\top$ and $\operatorname{PE}(b+l) = \operatorname{PE}(b)(\mathbf M^{\text{col}}_l)^\top$ with each $\mathbf M$ a block-diagonal rotation independent of position. Because the halves occupy disjoint channel blocks, the joint map is the block-diagonal $\mathbf M_{k,l} = \operatorname{diag}(\mathbf M^{\text{row}}_k, \mathbf M^{\text{col}}_l)$. For the inner product, the corollary of <XRef id="thm-tf1-pe-rotation" /> makes each frequency pair contribute $\cos(\omega_i\cdot\text{offset})$, so summing the $d/4$ row pairs and the $d/4$ column pairs,
$$
\operatorname{PE}_{\text{2D}}(a,b)\cdot\operatorname{PE}_{\text{2D}}(a+k,b+l) = \sum_{i=0}^{d/4-1}\cos(\omega_i k) + \sum_{i=0}^{d/4-1}\cos(\omega_i l). \qquad\blacksquare
$$

</SolutionBox>
