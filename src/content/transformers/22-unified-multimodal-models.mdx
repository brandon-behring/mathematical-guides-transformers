**Where we are.** Connectors permit continuous visual conditioning, while vector quantization yields discrete image symbols. A **unified multimodal model** places text and image units in one interleaved causal stream and runs a single decoder body across both. Modality-specific embeddings and output heads handle symbols at the boundaries; the chain rule supplies cross-modal conditionals for any chosen serialization. We then reduce a merged vocabulary to an ordinary autoregressive model, explain why finite categorical generation needs a discrete alphabet while conditioning may remain continuous, and define a mixed block-causal mask that permits bidirectional attention only inside fully observed image spans.

## The interleaved sequence

The object is one stream carrying both modalities. A unit is a symbol together with a tag saying which alphabet it came from; a document is any interleaving of text spans and image-code spans — a caption followed by its image, a page of text with figures, a dialogue with pictures.

<Theorem kind="definition" name="Interleaved multimodal sequence" id="def-interleaved">
Fix a text vocabulary $\mathcal V_{\mathrm{txt}}$ and an image-code vocabulary $\mathcal V_{\mathrm{img}}=\{1,\dots,K\}$ (the visual codebook of <XRef id="def-codebook" />). A **unit** is a pair $u_t=(c_t,x_t)$ with **modality tag** $c_t\in\{\mathrm{txt},\mathrm{img}\}$ and symbol $x_t\in\mathcal V_{c_t}$. An **interleaved sequence** $u_1,\dots,u_n$ is any concatenation of text and image-code spans in one stream. The model places a joint distribution on such sequences by the chain rule (<XRef id="def-autoregressive-lm" />),
$$
p_\theta(u_1,\dots,u_n) = \prod_{t=1}^{n} p_\theta(u_t\mid u_{<t}).
$$
<Sidenote>Early-fusion unified models of this form include CM3, Unified-IO, and Chameleon <Cite key="aghajanyan2022cm3" /><Cite key="lu2022unifiedio" /><Cite key="chameleonteam2024" />. Emu <Cite key="sun2023emu" /> and Transfusion <Cite key="zhou2024transfusion" /> keep the single interleaved stream but replace the categorical image factor with a *continuous* one (regressed embeddings; a diffusion head).</Sidenote>
</Theorem>

## Modality at the two interfaces

Modality is explicit at the input and output interfaces. Reading uses a per-modality embedding table; writing uses a per-modality head. Between those interfaces, the decoder shares one parameterized operator stack across modalities: it has no modality-specific branch, although modality information remains present in the embeddings and may also affect the attention mask.

<Theorem kind="definition" name="Modality-specific embedding" id="def-modality-embedding">
Each modality carries its own embedding table, $\mathbf E_{\mathrm{txt}}\in\R^{|\mathcal V_{\mathrm{txt}}|\times d}$ and $\mathbf E_{\mathrm{img}}\in\R^{K\times d}$, and a learned **modality offset** $\mathbf m_c\in\R^{d}$. Unit $u_t=(c_t,x_t)$ is embedded by
$$
\mathbf x_t \defeq \mathbf E_{c_t}[x_t,:] + \mathbf m_{c_t} + \mathbf P[t,:],
$$
the modality table lookup plus the offset plus a position code $\mathbf P[t,:]$ (<XRef id="def-sinusoidal-pe" />). This is the token embedding of <XRef id="def-token-embedding" /> with the single table replaced by a modality-indexed one — an instance of the embedding interface (<XRef id="def-embedding-interface" />).
</Theorem>

<Theorem kind="definition" name="Routed generation heads" id="def-routed-heads">
At each position the model produces a hidden state $\mathbf h_t$ and emits next-symbol logits through the head of the *predicted next modality*; the tag $c_{t+1}$ itself is predicted as part of the sequence (in practice via sentinel begin- and end-image tokens in the merged vocabulary, or a small tag head), supplying the factor $p_\theta(c_{t+1}\mid u_{\le t})$ that completes the chain rule of <XRef id="def-interleaved" />. With unembeddings $\mathbf W^{\mathrm{out}}_{\mathrm{txt}}\in\R^{d\times|\mathcal V_{\mathrm{txt}}|}$ and $\mathbf W^{\mathrm{out}}_{\mathrm{img}}\in\R^{d\times K}$,
$$
\begin{aligned}
p_\theta(u_{t+1}\mid u_{\le t})
&=p_\theta(c_{t+1}\mid u_{\le t})\,
  p_\theta(x_{t+1}\mid u_{\le t},c_{t+1}),\\
p_\theta(x_{t+1}\mid u_{\le t}, c_{t+1})
&= \softmax\!\big(\mathbf h_t \mathbf W^{\mathrm{out}}_{c_{t+1}}\big)_{x_{t+1}}.
\end{aligned}
$$
The first factor predicts the route; the second is a softmax over the selected modality's alphabet. Choosing which symbol head to apply is **modality routing**. If decoding constrains a task serialization, the next tag is externally fixed and its route is a point mass; otherwise the model generates the tag before routing.
</Theorem>

## Any-to-any: one body, every conditional

Because the joint is the chain-rule product, conditioning on any prefix and generating any suffix uses the very same factors. Splitting the stream differently into "observed" and "to-generate" gives captioning, text-to-image, or free interleaved generation — all from one trained model.

<Theorem kind="theorem" name="A unified body realizes every cross-modal conditional" id="thm-any-to-any">
Let $f_\theta$ be a single decoder-only body over interleaved sequences (<XRef id="def-interleaved" />). For any split of a sequence into an observed prefix $u_{\le s}$ and a generated suffix $u_{>s}$,
$$
p_\theta(u_{>s}\mid u_{\le s})
= \prod_{t>s} p_\theta(u_t\mid u_{<t})
= \prod_{t>s} p_\theta(c_t\mid u_{<t})\,
                     p_\theta(x_t\mid u_{<t},c_t),
$$
and these are the same tag and routed-symbol factors the model already computes. Choosing a serialization for the prefix and suffix instantiates **image-to-text** (observe image codes, generate text), **text-to-image** (observe text, generate image codes), and **interleaved** generation — all from the one body $f_\theta$, with no task-specific parameters. A serialization may prescribe the modality tags (making their factors point masses) or let the model generate them; in either case the factorization includes them.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof: split the chain-rule product at the end of the observed prefix. Which factors cancel in the conditional ratio, and which remain for generation?</MarginNote>

<Theorem kind="proof">
**Proof.**
By the chain rule (<XRef id="def-autoregressive-lm" />) and $u_t=(c_t,x_t)$, each joint factor splits as
$$
p_\theta(u_t\mid u_{<t})
=p_\theta(c_t\mid u_{<t})\,p_\theta(x_t\mid u_{<t},c_t).
$$
Thus $p_\theta(u_{1:n})=\prod_{t=1}^n p_\theta(c_t\mid u_{<t})p_\theta(x_t\mid u_{<t},c_t)$ for the fixed left-to-right order. Conditioning on any prefix with $p_\theta(u_{\le s})>0$ and marginalizing nothing (the suffix is fully generated) gives
$$
p_\theta(u_{>s}\mid u_{\le s})
= \frac{\prod_{t=1}^{n}p_\theta(c_t\mid u_{<t})p_\theta(x_t\mid u_{<t},c_t)}
        {\prod_{t=1}^{s}p_\theta(c_t\mid u_{<t})p_\theta(x_t\mid u_{<t},c_t)}
= \prod_{t=s+1}^{n}p_\theta(c_t\mid u_{<t})p_\theta(x_t\mid u_{<t},c_t),
$$
the trailing factors of the same product. At each generated position, the tag factor selects the route and the symbol factor is the selected head's softmax (<XRef id="def-routed-heads" />); both belong to the output interface. Modality affects the input vectors, may affect the attention mask, and selects the output route, but it does not select modality-specific body weights or operators. Hence assigning the prefix to one modality and the suffix to another yields a valid conditional, in the fixed serialization order, realized by evaluating $f_\theta$ with those positions observed; image-to-text and text-to-image are two of the four ordered choices of (prefix modality, suffix modality) — the other two are the same-modality continuations — and interleaved generation is the general mixed case with no single modality per side. A conditional in the opposite order (generating a prefix from a later suffix) is obtained by presenting that serialization at training time. $\qquad\blacksquare$
</Theorem>

## The merged-vocabulary reduction

The cleanest unified model erases explicit output-head routing: merge the two alphabets into one vocabulary, stack the two tables into one, and use a single head over the union. The vocabulary partition and any mixed attention mask still record modality or span structure, but the output interface is one softmax. The result is, symbol for symbol, the autoregressive transformer of <XRef id="def-autoregressive-lm" />.

<Theorem kind="proposition" name="A merged-vocabulary unified model is an autoregressive transformer" id="prop-merged-vocab">
Let $\mathcal V = \mathcal V_{\mathrm{txt}}\sqcup\mathcal V_{\mathrm{img}}$ be the disjoint union, of size $|\mathcal V|=|\mathcal V_{\mathrm{txt}}|+K$, with the bijection $\iota$ sending a text token $x$ to itself and an image code $z$ to $|\mathcal V_{\mathrm{txt}}|+z$. Stack the embeddings into $\mathbf E\in\R^{|\mathcal V|\times d}$ (row $\iota(c,x)$ equal to $\mathbf E_{c}[x,:]+\mathbf m_c$) and the unembeddings into $\mathbf W^{\mathrm{out}}\in\R^{d\times|\mathcal V|}$. Then the modality-specific embedding (<XRef id="def-modality-embedding" />) is a single lookup in $\mathbf E$ — the token embedding of <XRef id="def-token-embedding" /> — and a single shared head $\softmax(\mathbf h_t\mathbf W^{\mathrm{out}})$ over $\mathcal V$ is the autoregressive transformer of <XRef id="def-autoregressive-lm" /> with $|\mathcal V|=|\mathcal V_{\mathrm{txt}}|+K$. A unified model with merged vocabulary and shared head is therefore *exactly* that autoregressive model over a joint image–text alphabet.
</Theorem>

<Theorem kind="proof">
**Proof.**
The map $\iota:(c,x)\mapsto\iota(c,x)$ is a bijection from tagged units $\{(\mathrm{txt},x):x\in\mathcal V_{\mathrm{txt}}\}\cup\{(\mathrm{img},z):z\in\mathcal V_{\mathrm{img}}\}$ onto $\{1,\dots,|\mathcal V|\}$, because the two ranges $\{1,\dots,|\mathcal V_{\mathrm{txt}}|\}$ and $\{|\mathcal V_{\mathrm{txt}}|+1,\dots,|\mathcal V_{\mathrm{txt}}|+K\}$ are disjoint and cover $\mathcal V$. Define $\mathbf E[\iota(c,x),:]\defeq\mathbf E_c[x,:]+\mathbf m_c$; then the embedding of <XRef id="def-modality-embedding" /> is $\mathbf x_t=\mathbf E[\iota(c_t,x_t),:]+\mathbf P[t,:]$, a single table lookup, which is <XRef id="def-token-embedding" /> over $\mathcal V$. For the head, set the columns of $\mathbf W^{\mathrm{out}}$ on the text block equal to $\mathbf W^{\mathrm{out}}_{\mathrm{txt}}$ and on the image block to $\mathbf W^{\mathrm{out}}_{\mathrm{img}}$; the single softmax $\softmax(\mathbf h_t\mathbf W^{\mathrm{out}})$ is then a categorical distribution over all $|\mathcal V|$ symbols, i.e. the next-token distribution of <XRef id="def-autoregressive-lm" /> with vocabulary $\mathcal V$. The body between embedding and head is the decoder-only stack (<XRef id="def-decoder-only" />), unchanged. Every component therefore coincides with <XRef id="def-autoregressive-lm" /> over a vocabulary of size $|\mathcal V_{\mathrm{txt}}|+K$; cross-entropy training uses <XRef id="def-ce-loss" /> on that merged alphabet.

The relationship to routing is exact but requires the route probability. Write $o_{c,x}$ for the merged logit of symbol $x$ in modality block $c$ and $Z_c\defeq\sum_{x\in\mathcal V_c}e^{o_{c,x}}$. The merged softmax factors as
$$
p(c\mid\mathbf h_t)=\frac{Z_c}{\sum_{c'}Z_{c'}},
\qquad
p(x\mid\mathbf h_t,c)=\frac{e^{o_{c,x}}}{Z_c}.
$$
Thus it has the routed form of <XRef id="def-routed-heads" /> when the tag factor is the block-mass probability. If a task serialization externally fixes $c$, evaluating only that block computes the merged model's conditional distribution $p(x\mid\mathbf h_t,c)$; it is computational routing, not an approximation that requires off-modality mass to vanish. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="tip" label="Why this is the punchline">If a single softmax over a merged vocabulary *is* the text transformer, then "training a multimodal generator" and "training a language model" are the same optimization — the dual-footing thesis at full strength.</MarginNote>

## Why finite-vector categorical generation is discretized but conditioning need not be

The merged-vocabulary picture needs both modalities discrete because its standard generation head is a finite vector followed by softmax. A categorical law can exist on a countably infinite alphabet, but representing it exactly requires infinitely many logits and normalizing an infinite series; it is not the finite-vector head implemented by a transformer. A continuous output instead requires a density (or another continuous generative mechanism), not a categorical softmax indexed by every point. Conditioning is different: a modality the model merely *reads* is never normalized over, so it can stay continuous. This asymmetry supports continuous visual inputs from <XRef id="def-connector" /> alongside finite-codebook outputs from <XRef id="def-codebook" />.

<Theorem kind="proposition" name="A finite-vector categorical head forces a finite output alphabet; conditioning does not" id="prop-why-discrete">
Let $\mathcal A$ be an output alphabet. For finite $\mathcal A$, the standard autoregressive head emits one logit vector $\mathbf o\in\R^{|\mathcal A|}$ and applies softmax (<XRef id="def-autoregressive-lm" />), with partition function $Z=\sum_{x\in\mathcal A}\exp(o_x)$. A categorical law can also exist on a countably infinite alphabet when $\sum_{i=1}^{\infty}\exp(o_i)<\infty$, but exact evaluation then requires an infinite logit sequence and an infinite-series normalizer, not a finite output vector or finite matrix head. On a continuous domain, a categorical vector is the wrong object: an energy model needs a density $p(\mathbf v)=\exp(f(\mathbf v))/\int\exp(f(\mathbf z))\,d\mathbf z$, whose integral must be finite and tractable (or it needs a different mechanism such as diffusion). Hence a modality generated by the transformer's **standard finite-vector categorical head** must use a finite discrete alphabet (the visual codebook of <XRef id="def-codebook" />); a continuous-output density, normalizing flow, or diffusion head, as in Transfusion <Cite key="zhou2024transfusion" />, escapes discretization by replacing that head and its ordinary categorical cross-entropy. A modality that is only **conditioned on** enters as keys and values in attention (<XRef id="def-cross-attention" />) or as prepended embeddings (<XRef id="def-connector" />) and is never normalized over its domain, so it may remain continuous. Modality routing (<XRef id="def-routed-heads" />) is the hard, tag-determined analogue of the soft learned gate of mixture-of-experts (<XRef id="def-moe" />): both select a sub-module per position, by a tag here and a top-$k$ gate there.
</Theorem>

<Theorem kind="proof">
**Proof.**

**Finite alphabet.** For finite $\mathcal A$, a logit vector $\mathbf o\in\R^{|\mathcal A|}$ has finitely many entries and $Z=\sum_{x\in\mathcal A}\exp(o_x)$ is finite and strictly positive, so $p(x)=\exp(o_x)/Z$ is normalized and the categorical cross-entropy of <XRef id="def-ce-loss" /> is directly computable.

**Countably infinite alphabet.** For $\mathcal A=\N$, the same formula defines a categorical law whenever the series $Z=\sum_{i=1}^{\infty}\exp(o_i)$ converges; mathematical existence therefore does **not** force finiteness. But exact representation needs one logit per $i$ and exact normalization needs the infinite sum, so it is not a finite-vector softmax $\softmax(\mathbf h\mathbf W^{\mathrm{out}})$.

**Continuous domain.** For $\mathcal A=\R^d$, a probability assigning mass through a finite categorical vector cannot cover the continuum; a density-based energy model instead uses $Z=\int_{\R^d}\exp(f(\mathbf v))\,d\mathbf v$, which defines a density only when the integral converges and is usable exactly only when that normalizer can be computed or otherwise handled. Thus the ordinary transformer head directly supports a finite discrete output, while a continuous generator must replace it with a suitable density or implicit mechanism — establishing the scoped output claim.

**Conditioning and routing.** For conditioning, a continuous observation is first mapped to finitely many vectors. Projections, attention, residual paths, normalization, and feed-forward layers then process those vectors through deterministic finite-dimensional maps; no step defines a probability normalized over all possible input observations. Continuous inputs are therefore admissible — the input claim. Finally, write the modality tag $c_{t+1}$ as a one-hot gate over the two heads; the routed head $\mathbf W^{\mathrm{out}}_{c_{t+1}}$ is the gate-selected expert. Mixture-of-experts (<XRef id="def-moe" />) selects expert FFNs by a learned top-$k$ gate; replacing the learned top-$k$ gate by the hard one-hot modality tag gives modality routing, so the two share the select-a-sub-module structure with a learned versus prescribed gate. $\qquad\blacksquare$
</Theorem>

## The mixed block-causal mask

A token-wise autoregressive unified model is causal at every generated position — each target token may use only observed context and earlier target tokens. An image span that is **fully supplied as context**, however, has no unknown token to leak, so its positions may attend to one another in both directions. The mixed mask combines these regimes without granting a generated image token access to later target image codes.

<Theorem kind="definition" name="Mixed block-causal mask" id="def-mixed-mask">
For a particular conditional-generation example, partition the interleaved sequence into contiguous **segments** (maximal same-modality spans), and let $\mathcal B_{\mathrm{ctx}}$ be the image segments whose entire contents are supplied as observed context rather than predicted by the token-wise autoregressive objective. The **mixed block-causal mask** allows position $i$ to attend to position $j$ iff
$$
M_{ij}=1 \iff \underbrace{j\le i}_{\text{causal}} \ \ \text{or}\ \ \underbrace{\text{$i,j\in B$ for some $B\in\mathcal B_{\mathrm{ctx}}$}}_{\text{fully observed image context}}.
$$
Causality is enforced across segments. Positions inside a fully observed context image may attend bidirectionally, because all of them are already given; positions in a target image segment use only the first clause and remain causal in rasterized token order. When the stream is all text, $\mathcal B_{\mathrm{ctx}}$ is empty, so $M$ is the causal mask of <XRef id="def-causal-mask" />; the unified model then reduces to the text decoder-only transformer.
</Theorem>

<Theorem kind="remark" name="The body is the proof of the thesis" id="rem-body-unchanged">
Across all three designs of the figure — merged single-stream, two-path, and continuous-input, discrete-output — the transformer body uses the same decoder-only block ingredients and stacking as the text model (<XRef id="def-decoder-only" />). What differs is the embedding, the head, and the mask. This architectural reuse does **not** automatically transfer an unmasked universal-approximation theorem such as <XRef id="thm-universal-approx" />: causal and mixed masks restrict information flow, so an approximation guarantee for those models requires a separate mask-compatible theorem. The dual-footing claim here is therefore architectural and algebraic, not an unproved universality transfer: the same sequence-processing blocks accept image and text representations in $\R^{n\times d}$.
</Theorem>

**Exercise 22.1 (Prove).** Prove the any-to-any factorization (<XRef id="thm-any-to-any" />) for the special case of image captioning: with the observed prefix the image codes $u_{\le s}$ and the generated suffix the caption tokens $u_{>s}$, show the conditional is the trailing product of next-token factors, and identify the modality-dependent components — and why there are exactly two.

<SolutionBox exercise="22.1">

By the chain rule and $u_t=(c_t,x_t)$,
$$
p_\theta(u_{1:n})=\prod_{t=1}^n p_\theta(c_t\mid u_{<t})p_\theta(x_t\mid u_{<t},c_t).
$$
Dividing by the corresponding product through $s$ cancels the prefix and leaves
$$
p_\theta(u_{>s}\mid u_{\le s})
=\prod_{t=s+1}^{n}p_\theta(c_t\mid u_{<t})p_\theta(x_t\mid u_{<t},c_t),
$$
the caption generated left to right conditioned on the image codes. At each caption position the first factor predicts (or, under a fixed caption serialization, deterministically supplies) the text tag, and the second comes from the routed text head $\mathbf W^{\mathrm{out}}_{\mathrm{txt}}$ over $\mathcal V_{\mathrm{txt}}$ (<XRef id="def-routed-heads" />). The body uses shared weights across modalities. There are still exactly two modality-dependent **interfaces**: the input embedding reads the image-tagged prefix, while the output interface contains tag prediction, routing, and the selected symbol head; the tag factor is part of that output interface, not a third place where modality enters the body. $\qquad\blacksquare$

</SolutionBox>

**Exercise 22.2 (Prove).** Prove the merged-vocabulary reduction (<XRef id="prop-merged-vocab" />) by exhibiting the bijection between tagged units and merged indices and checking that the stacked embedding and head reproduce <XRef id="def-token-embedding" /> and <XRef id="def-autoregressive-lm" />.

<SolutionBox exercise="22.2">

Define $\iota(\mathrm{txt},x)=x$ for $x\in\{1,\dots,|\mathcal V_{\mathrm{txt}}|\}$ and $\iota(\mathrm{img},z)=|\mathcal V_{\mathrm{txt}}|+z$ for $z\in\{1,\dots,K\}$. The two image and text ranges are disjoint and their union is $\{1,\dots,|\mathcal V_{\mathrm{txt}}|+K\}=\mathcal V$, so $\iota$ is a bijection. Set $\mathbf E[\iota(c,x),:]\defeq\mathbf E_c[x,:]+\mathbf m_c$: then the modality embedding $\mathbf E_{c_t}[x_t,:]+\mathbf m_{c_t}+\mathbf P[t,:]$ equals $\mathbf E[\iota(c_t,x_t),:]+\mathbf P[t,:]$, one lookup in $\mathbf E$ — precisely <XRef id="def-token-embedding" /> over $\mathcal V$. Set $\mathbf W^{\mathrm{out}}$'s text and image column blocks to $\mathbf W^{\mathrm{out}}_{\mathrm{txt}}$ and $\mathbf W^{\mathrm{out}}_{\mathrm{img}}$; then $\softmax(\mathbf h_t\mathbf W^{\mathrm{out}})$ is a categorical over $\mathcal V$, the next-token head of <XRef id="def-autoregressive-lm" />. The body is unchanged, so the unified model is <XRef id="def-autoregressive-lm" /> with $|\mathcal V|=|\mathcal V_{\mathrm{txt}}|+K$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 22.3 (Prove).** Prove the scoped output claim of <XRef id="prop-why-discrete" /> by comparing (i) a finite alphabet, (ii) a countably infinite alphabet, and (iii) a continuous domain. Give each normalizer and distinguish mathematical existence from what a finite-vector transformer head can represent and normalize exactly.

<SolutionBox exercise="22.3">

A categorical head sets $p(x)=\exp(o_x)/Z$. For finite $\mathcal A$, $Z=\sum_{x\in\mathcal A}\exp(o_x)$ is a finite, strictly positive sum, so a finite logit vector gives a normalized distribution and $-\log p(x)$ (categorical cross-entropy, <XRef id="def-ce-loss" />) is directly computable. For $\mathcal A=\N$, $Z=\sum_{i=1}^{\infty}\exp(o_i)$ may converge — for example, logits $o_i=-i$ give a geometric series — so a countably infinite categorical distribution can exist. It nevertheless requires infinitely many logits and exact evaluation of an infinite sum, neither of which is a finite matrix readout followed by vector softmax. For $\mathcal A=\R^d$, a continuous model needs a density with $Z=\int_{\R^d}\exp(f(\mathbf v))\,d\mathbf v$; this is not a categorical vector, and it is valid only when the integral is finite (with exact likelihood additionally requiring a tractable normalizer). Thus the standard finite-vector categorical transformer head requires a finite discrete alphabet, exactly what the visual codebook supplies (<XRef id="def-codebook" />); the broader mathematical class of categorical distributions need not have finite support. $\qquad\blacksquare$

</SolutionBox>

**Exercise 22.4 (Construct).** Describe a continuous-input, discrete-output unified model: which interface (Ch 20 connector or Ch 21 codebook) handles the conditioned image and which handles the generated image, and justify each choice using <XRef id="prop-why-discrete" />.

<SolutionBox exercise="22.4">

Conditioned images enter through a **connector** (<XRef id="def-connector" />) as continuous soft tokens: they are only read — used as prepended embeddings or cross-attention keys and values — and never normalized over, so by the input half of <XRef id="prop-why-discrete" /> they may stay continuous, avoiding the lossy quantization of <XRef id="prop-rate" />. Generated images use the **discrete codebook** (<XRef id="def-codebook" />): generation runs through the transformer's standard finite-vector categorical head, which by the output half of <XRef id="prop-why-discrete" /> directly supports a finite alphabet, so the image is emitted as code indices and decoded by $\Psi$. A countably infinite categorical law is mathematically possible but is not this finite head, while continuous generation would require replacing it with a density or implicit generator. The design therefore conditions with the higher-fidelity continuous representation and generates with the finite discrete representation the chosen head supports. $\qquad\blacksquare$

</SolutionBox>

**Exercise 22.5 (Prove).** Show the mixed block-causal mask (<XRef id="def-mixed-mask" />) reduces to the causal mask of <XRef id="def-causal-mask" /> when the stream contains no observed context image segments. Explain why bidirectionality inside a fully observed image context does not leak a target, and why an image segment generated by the chapter's token-wise autoregressive factorization must remain causal.

<SolutionBox exercise="22.5">

If there are no fully observed context image segments, $\mathcal B_{\mathrm{ctx}}=\varnothing$, so the second clause is never satisfied and $M_{ij}=1\iff j\le i$, exactly the lower-triangular causal mask of <XRef id="def-causal-mask" />. If $B\in\mathcal B_{\mathrm{ctx}}$, every token in $B$ is supplied before target scoring, so letting rows in $B$ see all other columns in $B$ reveals no unknown target; the clause also adds no edge from $B$ into a later segment. By contrast, for a generated image segment the token-wise factorization contains factors $p_\theta(u_t\mid u_{<t})$. Allowing its row $t$ to see a later target $u_j$, $j>t$, would condition that factor on the answer and invalidate the factorization. Such a segment is therefore excluded from $\mathcal B_{\mathrm{ctx}}$ and remains causal. $\qquad\blacksquare$

</SolutionBox>
