**Where we are.** A vision encoder emits contextual feature rows, while a decoder expects model-width embeddings. A **connector** maps between those interfaces. We compare position-wise projectors with learned-query resamplers that compress a variable-length visual sequence into a fixed number of soft tokens. We prove projected-value convexity for the single-layer output and order invariance without positional inputs, then account for compute and cache savings. A conditional data-processing bound states what the downstream model cannot recover once the connector has discarded information. Finally, gated cross-attention offers another coupling: a closed gate preserves the pretrained language model at initialization, while training can open a visual path where it is useful.

## The connector as an embedding interface

The picture to hold is a pipeline with a seam. On the left, a frozen vision encoder has already produced $n$ feature vectors of width $d_{\mathrm{vis}}$. On the right, a language model expects embeddings of width $d$. The connector is exactly the adapter that crosses this seam, and crossing it is nothing more than a fresh instance of the embedding map of <XRef id="def-embedding-interface" />: a unit — here a visual feature vector rather than a token id — is sent to a vector in $\R^{d}$ and dropped into the sequence the transformer reads.

<Theorem kind="definition" name="Connector" id="def-connector">
Let a frozen vision encoder produce visual features $\mathbf V \in \R^{n\times d_{\mathrm{vis}}}$ with rows $\mathbf v_1,\dots,\mathbf v_n$ (one per patch). A **connector** is a learned map
$$
g_\phi:\R^{n\times d_{\mathrm{vis}}}\longrightarrow \R^{m\times d},
\qquad
\mathbf S = g_\phi(\mathbf V),
$$
producing $m$ **soft visual tokens** — rows of $\mathbf S$ living in the language model's embedding space $\R^{d}$. These rows are concatenated with the text token embeddings and fed to the language model as a single sequence,
$$
\hat{\mathbf X} = \big(\,\underbrace{\mathbf s_1;\dots;\mathbf s_m}_{\text{visual}}\ ;\ \underbrace{\Emb(x_1);\dots;\Emb(x_T)}_{\text{text}}\,\big)\in\R^{(m+T)\times d},
$$
so the soft tokens enter through the embedding interface (<XRef id="def-embedding-interface" />) exactly as text embeddings do.<Sidenote>"Soft" because the rows are arbitrary vectors in $\R^d$, not lookups from a finite vocabulary; <XRef id="def-codebook-embedding" /> defines the discrete alternative.</Sidenote>
</Theorem>

<MarginNote variant="pattern" label="Interface placement">A connector occupies the embedding boundary; gated cross-attention occupies the coupling boundary. Between them, the decoder core remains modality-blind in the sense of <XRef id="rem-modality-blind" />.</MarginNote>

Two design choices now fork. Either keep every patch — map each visual feature to one soft token and let the language model attend over all $n$ of them — or compress, summarizing $n$ patches into a smaller fixed number $m$ of soft tokens before the language model ever sees them. The first is the projector; the second is the resampler.

## The projector

The simplest connector is position-wise: apply the *same* small network to each visual feature independently, with no mixing across patches. There is one soft token per patch, so $m=n$, and the only thing learned is the per-position map from $\R^{d_{\mathrm{vis}}}$ to $\R^{d}$.

<Theorem kind="definition" name="Projector" id="def-projector">
A **projector** is a connector applied row-wise by a shared map $h_\phi:\R^{d_{\mathrm{vis}}}\to\R^{d}$,
$$
g_\phi(\mathbf V)_i = h_\phi(\mathbf v_i),\qquad i=1,\dots,n,\qquad m=n,
$$
with no interaction between positions. The two standard instances are the **linear projector** $h_\phi(\mathbf v)=\mathbf v\,\mathbf W + \mathbf b$ with $\mathbf W\in\R^{d_{\mathrm{vis}}\times d}$, and the **MLP projector** $h_\phi(\mathbf v)=\sigma(\mathbf v\,\mathbf W_1 + \mathbf b_1)\mathbf W_2 + \mathbf b_2$ with a pointwise nonlinearity $\sigma$.<Sidenote>The linear projector is the original LLaVA connector <Cite key="liu2023llava" />; the two-layer MLP is its LLaVA-1.5 refinement <Cite key="liu2023improvedllava" />. Both are deliberately the lightest possible bridge, pushing all alignment work onto the language model.</Sidenote>
</Theorem>

A projector preserves the patch grid one-for-one: the language model receives $n$ projected visual tokens and may attend over all of them at every layer. It performs no cross-patch pooling before the model decides what matters, but its position-wise map need not be injective, so preserving one row per patch does not guarantee that all visual information survives. The design is expensive because those $n$ tokens occupy the sequence (and the KV cache, <XRef id="def-kv-cache" />) at every one of the language model's layers and decoding steps. When $n$ is large (a high-resolution image can produce many hundreds of patches), one looks for a connector that hands the language model only a handful of tokens.

## The learned-query resampler

The resampler replaces "one token per patch" with "a fixed budget of tokens, filled by attention." It carries a small set of $m$ *learned* query vectors — parameters, not functions of the input — and lets them attend over the visual features. Each query pulls a weighted summary of the patches; the $m$ summaries are the soft tokens. Because the queries are fixed in number, the output length is $m$ no matter how many patches come in.

<Theorem kind="definition" name="Learned-query resampler" id="def-resampler">
Fix $m$ **latent queries** $\mathbf Q_{\mathrm{lat}}\in\R^{m\times d}$, learned parameters independent of the input. A (single-layer) **resampler** is the cross-attention map (<XRef id="def-cross-attention" />) of the latent queries against the visual features as keys and values,
$$
\begin{aligned}
g_\phi(\mathbf V)
&= \operatorname{CA}(\mathbf Q_{\mathrm{lat}},\mathbf V)\\
&=\mathbf A\,(\mathbf V\mathbf W^V)\mathbf W^O
=\mathbf A\,(\mathbf V\mathbf W^V\mathbf W^O).
\end{aligned}
$$
The row-wise attention matrix $\mathbf A\in\R^{m\times n}$ is
$$
\mathbf A = \softmax\!\left(
  \frac{(\mathbf Q_{\mathrm{lat}}\mathbf W^Q)
  (\mathbf V\mathbf W^K)^{\top}}{\sqrt{d_k}}
\right).
$$
Here $\mathbf W^Q\in\R^{d\times d_k}$, $\mathbf W^K\in\R^{d_{\mathrm{vis}}\times d_k}$, $\mathbf W^V\in\R^{d_{\mathrm{vis}}\times d_v}$, and $\mathbf W^O\in\R^{d_v\times d}$; $\operatorname{CA}$ is exactly the cross-attention of <XRef id="def-cross-attention" /> with *parameter* queries. A deep resampler stacks $L_r$ such blocks (each a cross-attention plus a feed-forward sublayer); the output is always $m$ rows.<Sidenote>The learned-query cross-attention resampler is the Perceiver's mechanism <Cite key="jaegle2021perceiver" />, the Flamingo Perceiver Resampler <Cite key="alayrac2022flamingo" />, and the BLIP-2 Q-Former <Cite key="li2023blip2" />.</Sidenote>
</Theorem>

The funnel is the whole point: the visual side may grow without bound, while the language side always receives $m$ tokens. We now make precise what those $m$ tokens can and cannot be.

<Theorem kind="proposition" name="Resampler outputs are fixed in number, convex, and order-invariant" id="prop-resampler-convex">
For the single-layer attention output of <XRef id="def-resampler" />, with output-projected values $\tilde{\mathbf v}_j \defeq \mathbf v_j\mathbf W^V\mathbf W^O\in\R^d$:
1. **Fixed length.** The output has exactly $m$ rows for every $n$.
2. **Convexity.** Each output row $g_\phi(\mathbf V)_i = \sum_{j=1}^{n} A_{ij}\,\tilde{\mathbf v}_j$ is a convex combination of the projected values $\{\tilde{\mathbf v}_j\}$, hence lies in their convex hull (<XRef id="prop-row-stochastic-convex" />). This is a hull constraint in the output space **after** $\mathbf W^V\mathbf W^O$; the learned projections may move those values outside the raw-input hull.
3. **Order invariance.** If no positional information is added to $\mathbf V$, then permuting the patches leaves every output row unchanged: the resampler is invariant to patch order.
</Theorem>

<Theorem kind="proof">
**Proof.**
(1) The query matrix $\mathbf Q_{\mathrm{lat}}$ has $m$ rows, so the attention matrix $\mathbf A$ is $m\times n$ and $\mathbf A(\mathbf V\mathbf W^V)\mathbf W^O$ is $m\times d$, independent of $n$. (2) By associativity and distributivity, $\mathbf A(\mathbf V\mathbf W^V)\mathbf W^O=\mathbf A(\mathbf V\mathbf W^V\mathbf W^O)$. Each row $\mathbf A_{i,:}$ is a softmax output, hence nonnegative with $\sum_j A_{ij}=1$; therefore $g_\phi(\mathbf V)_i=\sum_j A_{ij}\tilde{\mathbf v}_j$ is a convex combination of the $\tilde{\mathbf v}_j$ and lies in their convex hull by <XRef id="prop-row-stochastic-convex" />. (3) Let $\pi$ be a permutation of $\{1,\dots,n\}$ acting on the rows of $\mathbf V$. Because the keys and values are computed row-wise, $\mathbf v_j\mapsto\mathbf v_{\pi(j)}$ sends $A_{ij}\mapsto A_{i\pi(j)}$ and $\tilde{\mathbf v}_j\mapsto\tilde{\mathbf v}_{\pi(j)}$; the score $\inner{\mathbf q_i\mathbf W^Q}{\mathbf v_{\pi(j)}\mathbf W^K}$ for fixed query $i$ depends on the patch only through its own content, so reindexing $j'=\pi(j)$ gives $\sum_{j}A_{i\pi(j)}\tilde{\mathbf v}_{\pi(j)}=\sum_{j'}A_{ij'}\tilde{\mathbf v}_{j'}$, the original row. Hence each output row is unchanged. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="tip" label="Order invariance cuts both ways">Invariance to patch order is why resamplers add 2D positions (<XRef id="def-pe-2d" />) to $\mathbf V$ when *where* a feature sits matters. Without them a resampler sees a bag of patches.</MarginNote>

The convexity clause is the single attention layer's analogue of the pooling and cross-attention bounds proved earlier: a softmax-weighted readout stays in the hull of the output-projected values. It does not constrain the learned projections relative to the raw visual-feature hull. A deep resampler retains fixed length and, without positions, order invariance, but its residual and nonlinear feed-forward sublayers need not retain this one-layer hull constraint. The cost clause is what makes the design worthwhile.

<Theorem kind="proposition" name="Resampler cost and cache savings" id="prop-resampler-cost">
A resampler of $L_r$ layers with $m$ queries over $n$ patches at width $d$ costs
$$
\Theta\!\big(L_r[\,mnd+(m+n)d^2\,]\big)
$$
arithmetic: $\Theta(mnd)$ for scores and value aggregation, $\Theta(nd^2)$ for key and value projections, and $\Theta(md^2)$ for latent-query and output projections and the latent feed-forward sublayer. In the intended regime $m=O(n)$ this simplifies to $\Theta(L_r\,n\,d\,(m+d))$. The cost is paid **once** when the image is encoded. Downstream, the language model of depth $N$ holds the connector's output in its KV cache (<XRef id="def-kv-cache" />, <XRef id="prop-kv-memory" />), costing $\Theta(N\,m\,d)$ memory for the visual tokens. Replacing a projector ($m=n$) by a resampler with $m\ll n$ therefore shrinks the visual cache by the factor $n/m$ and removes $\Theta(n-m)$ tokens from every language-model layer and decode step. The trade is favorable when the language model is deep (large $N$) or autoregressive decoding is long, and $n\gg m$.
</Theorem>

<Theorem kind="proof">
**Proof.**
One resampler layer forms $\mathbf A\in\R^{m\times n}$ — an $m\times n$ array whose entries are inner products of width $d_k=\Theta(d)$ — and aggregates projected values, for $\Theta(mnd)$ work. The key and value projections of $n$ visual rows cost $\Theta(nd^2)$. Projecting the $m$ latent queries and outputs and applying the width-$\Theta(d)$ feed-forward sublayer to those $m$ rows cost $\Theta(md^2)$. Summing gives $\Theta(mnd+(m+n)d^2)$ per layer and the displayed $L_r$-layer cost; when $m=O(n)$, $(m+n)d^2=\Theta(nd^2)$, yielding $\Theta(L_rnd(m+d))$. The cache claim is <XRef id="prop-kv-memory" /> with sequence length equal to the number of visual tokens: $m$ for the resampler versus $n$ for the projector, over $N$ layers, giving $\Theta(Nmd)$ versus $\Theta(Nnd)$ and a ratio $n/m$. The per-layer and per-step token counts drop by $n-m$ identically. $\qquad\blacksquare$
</Theorem>

## The connector is an information bottleneck

Length reduction alone is not an information-loss theorem for real-valued soft tokens: on a restricted data support, a continuous representation with $m<n$ may still be injective, and ideal real numbers have no finite bit budget. The rigorous statement begins once the connector is non-injective or its capacity is explicitly bounded (for example, by a discrete alphabet or finite precision). In every case, conditional data processing says downstream computation cannot create new image information beyond what the connector retained, after accounting for text supplied separately.

<Theorem kind="definition" name="Mutual information" id="def-mutual-information">
For random variables $X,Z$, the **mutual information** is $I(X;Z) \defeq H(X) - H(X\mid Z)$, the reduction in uncertainty about $X$ from observing $Z$; conditioned on a third variable $C$, $I(X;Z\mid C)\defeq H(X\mid C)-H(X\mid Z,C)$. Mutual information is symmetric and nonnegative, and satisfies $I(X;Z)\le H(Z)$ when $Z$ is discrete, with $H$ the (Shannon) entropy — <XRef id="def-entropy" /> extended in the standard way to joint distributions, with conditional entropy $H(X\mid Z) \defeq \E_{z}\big[H(X\mid Z=z)\big]$; for continuous $X$ read all statements over a fine discretization of $X$, which is all the bound below uses.<Sidenote>The information-bottleneck reading of a learned representation is due to <Cite key="tishby1999information" /> and was applied to deep networks by <Cite key="tishby2015deeplearning" />.</Sidenote>
</Theorem>

<Theorem kind="proposition" name="Connector capacity bounds image information (data-processing)" id="prop-connector-data-processing-bound">
Let $X$ be the image (or the full visual features), $\mathbf S=g_\phi(\mathbf V)$ the connector output, $C$ the text or task context supplied separately, and $\hat Y$ the model's prediction. Assume the conditional Markov property $X\to\mathbf S\to\hat Y$ given $C$: once $(\mathbf S,C)$ are known, $\hat Y$ does not depend further on $X$. Then
$$
I(X;\hat Y\mid C)\;\le\; I(X;\mathbf S\mid C).
$$
If $C$ is fixed, this is the ordinary inequality $I(X;\hat Y)\le I(X;\mathbf S)$. If moreover $\mathbf S$ is discrete, then $I(X;\hat Y\mid C)\le I(X;\mathbf S\mid C)\le H(\mathbf S\mid C)$. In particular, $m$ symbols from a codebook of size $K$ give $H(\mathbf S\mid C)\le m\log_2K$ bits. This explicit capacity bound — not the inequality $m<n$ by itself for continuous soft tokens — limits the image information available beyond the context.
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the proof: draw the chain from image to connector output to prediction. Which conditional independence licenses data processing, and where does a discrete connector cap the retained entropy?</MarginNote>

<Theorem kind="proof">
**Proof.**
Expand conditional mutual information by the chain rule two ways:
$$
I(X;\mathbf S,\hat Y\mid C)
=I(X;\mathbf S\mid C)+I(X;\hat Y\mid\mathbf S,C)
=I(X;\hat Y\mid C)+I(X;\mathbf S\mid\hat Y,C).
$$
The conditional Markov property makes $I(X;\hat Y\mid\mathbf S,C)=0$, while conditional mutual information gives $I(X;\mathbf S\mid\hat Y,C)\ge0$. Therefore
$$
I(X;\mathbf S\mid C)=I(X;\hat Y\mid C)+I(X;\mathbf S\mid\hat Y,C)\ge I(X;\hat Y\mid C).
$$
For discrete $\mathbf S$, $I(X;\mathbf S\mid C)=H(\mathbf S\mid C)-H(\mathbf S\mid X,C)\le H(\mathbf S\mid C)$. A length-$m$ variable over $K$ symbols has at most $K^m$ outcomes and hence entropy at most $\log_2K^m=m\log_2K$. If $C$ is constant, deleting it from every conditioning set recovers the unconditional proof. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="formula" label="Why m matters">A discrete connector emitting $m$ tokens over a codebook of size $K$ carries at most $m\log_2 K$ bits about the image (<XRef id="def-codebook-embedding" />). Shrinking $m$ narrows the channel, so the bottleneck must become selective.</MarginNote>

The bound is the rigorous statement of the bottleneck intuition when the connector is non-injective or capacity-limited: what its output omits cannot be restored by the language model, however capable. Under such a budget, training the connector is the task of learning which image information is task-relevant.

## Gated cross-attention: coupling without disturbing the base model

The connectors above inject visual tokens at the embedding interface and otherwise leave the language model untouched. An alternative couples vision through new cross-attention blocks *inside* the language model (Interface (3), <XRef id="def-cross-attention" />). The danger is obvious: inserting fresh, randomly initialized blocks into a carefully pretrained — often frozen — language model would corrupt its behavior on the first forward pass. Flamingo's fix is a gate. Each inserted block is multiplied by $\tanh(\gamma)$ for a learnable scalar $\gamma$ initialized to zero, so the block contributes nothing at the start and the language model is preserved exactly.

<Theorem kind="definition" name="Gated cross-attention" id="def-gated-xattn">
Let $\mathbf h\in\R^{T\times d}$ be a language-model hidden state and $\mathbf V$ the visual features. A **gated cross-attention** layer augments the stream by
$$
\mathbf h \;\leftarrow\; \mathbf h + \tanh(\gamma)\,\operatorname{CA}(\mathbf h,\mathbf V),
$$
with a learnable **gate** $\gamma\in\R$ (initialized $\gamma=0$) and cross-attention $\operatorname{CA}$ (<XRef id="def-cross-attention" />) whose parameters we collect as $\theta$. A paired gated feed-forward sublayer is defined identically with its own gate.<Sidenote>The $\tanh$-gated cross-attention layers interleaved into a frozen language model are the Flamingo coupling <Cite key="alayrac2022flamingo" />.</Sidenote>
</Theorem>

  desc="The language-model residual stream enters a visual cross-attention branch. Its output passes through a scalar gate before residual addition. A zero gate removes the entire branch while leaving a gradient path that can open it during training."
  caption="Gated cross-attention scales the visual branch before adding it to the language-model residual stream. A zero-initialized gate begins closed, preserving the base model while allowing training to open the visual path."

<Theorem kind="theorem" name="Gated cross-attention is the identity at initialization" id="thm-gated-identity">
Let $f_{\mathrm{aug}}$ be the language model augmented with gated cross-attention layers (<XRef id="def-gated-xattn" />) and $f_{\mathrm{LM}}$ the base model. At initialization $\gamma=0$:
1. **Identity.** $f_{\mathrm{aug}} = f_{\mathrm{LM}}$ as functions — the augmented model computes exactly the base model's output, for every input.
2. **Gradient routing.** For any loss $\mathcal L$, the cross-attention parameters receive no gradient, $\nabla_\theta\mathcal L = \mathbf 0$, while the gate generically does, $\partial\mathcal L/\partial\gamma \ne 0$. The gate moves first; once $\gamma\ne 0$ the $\theta$-gradient is no longer forced to zero by the gate, and the visual path opens.
</Theorem>

<Theorem kind="proof">
**Proof.**
(1) Since $\tanh(0)=0$, the added term $\tanh(\gamma)\operatorname{CA}(\mathbf h,\mathbf V)=0$ at every gated layer, so each update is $\mathbf h\leftarrow\mathbf h$ and the network reduces to $f_{\mathrm{LM}}$ identically — independent of $\theta$ and of $\mathbf V$. (2) Write one gated layer's output as $\mathbf y = \mathbf h + \tanh(\gamma)\,c_\theta$ with $c_\theta\defeq\operatorname{CA}_\theta(\mathbf h,\mathbf V)$. Collect the parameters as $\theta\in\R^p$, set $\mathbf g_{\mathrm{vec}}\defeq\operatorname{vec}(\partial\mathcal L/\partial\mathbf y)\in\R^{Td}$, and define the Jacobian $\mathbf J_\theta\defeq\partial\operatorname{vec}(c_\theta)/\partial\theta^\top\in\R^{Td\times p}$ with Euclidean adjoint $\mathbf J_\theta^*=\mathbf J_\theta^\top$. The chain rule gives
$$
\nabla_\theta\mathcal L
= \tanh(\gamma)\,\mathbf J_\theta^*\mathbf g_{\mathrm{vec}},
\qquad
\frac{\partial\mathcal L}{\partial\gamma}
= \operatorname{sech}^2(\gamma)\,\inner{\mathbf g_{\mathrm{vec}}}{\operatorname{vec}(c_\theta)} .
$$
At $\gamma=0$ the first prefactor $\tanh(0)=0$ kills $\nabla_\theta\mathcal L$ regardless of $c_\theta$, while the second prefactor $\operatorname{sech}^2(0)=1$ leaves $\partial\mathcal L/\partial\gamma=\inner{\mathbf g_{\mathrm{vec}}}{\operatorname{vec}(c_\theta)}$, which is nonzero whenever the vectorized upstream gradient is not orthogonal to the (generically nonzero) vectorized cross-attention output. Thus the first optimization step moves $\gamma$ off zero but not $\theta$; for $\gamma\ne0$, $\tanh(\gamma)\ne0$, so the gate no longer forces $\nabla_\theta\mathcal L$ to vanish and it is generically nonzero. $\qquad\blacksquare$
</Theorem>

<Theorem kind="remark" name="The same trick as LoRA" id="rem-lora-analogy">
Identity-at-initialization is precisely the design of the low-rank adapter (<XRef id="def-lora" />): there the update $\mathbf B\mathbf A$ starts at zero because $\mathbf B$ is zero-initialized, so the adapted model equals the pretrained model until training pulls $\mathbf B$ off zero. Flamingo's $\tanh(\gamma)$ with $\gamma=0$ plays the role of the zero-initialized $\mathbf B$: a single learnable knob that begins closed, guarantees the powerful pretrained model is unharmed at step zero, and is opened gradually by gradient descent. Both are instances of *grafting new capacity onto a frozen network without a destructive cold start* — the gate, not the block, is what is learned first.
</Theorem>

## Exercises

Attempt each before expanding its solution.

**Exercise 20.1 (Prove).** Show that a linear projector (<XRef id="def-projector" />) is the special case of a connector with $m=n$ and no cross-patch interaction. Then compare it with a single-layer resampler in the **same projected output space**: fix $\tilde{\mathbf v}_j=\mathbf v_j\mathbf W^V\mathbf W^O$ and take the projector to use $\mathbf W=\mathbf W^V\mathbf W^O$ with zero bias. Where do the two outputs lie? Explain why neither statement implies confinement to the raw-input hull.

<SolutionBox exercise="20.1">

A linear projector sets $g_\phi(\mathbf V)_i=\mathbf v_i\mathbf W+\mathbf b$, a function of $\mathbf v_i$ alone, so $m=n$ and there is no cross-patch interaction. With $\mathbf W=\mathbf W^V\mathbf W^O$ and $\mathbf b=0$, its rows are exactly the projected values $\tilde{\mathbf v}_i$, hence points (and possible vertices) of $\operatorname{conv}\{\tilde{\mathbf v}_j\}$. The single-layer resampler outputs $\sum_jA_{ij}\tilde{\mathbf v}_j$ and therefore lies anywhere in that same projected-value hull (<XRef id="prop-resampler-convex" />), while using cross-patch mixing and possibly $m<n$ rows. The projection itself can move both kinds of output outside the raw-input hull: in one dimension, raw values $\{0,1\}$ with $W^V=2$ and $W^O=1$ become $\{0,2\}$. A nonzero projector bias or a different projection can also leave the fixed comparison hull, but then the two maps no longer share the same projected space. The meaningful contrast is row-wise mapping versus convex cross-patch pooling, not raw-hull superiority. $\qquad\blacksquare$

</SolutionBox>

**Exercise 20.2 (Prove).** Prove clause (3) of <XRef id="prop-resampler-convex" /> fails once a position-dependent vector is added to each visual feature: if $\mathbf v_j \mapsto \mathbf v_j + \mathbf p_j$ with distinct positions $\mathbf p_j$, exhibit a permutation that changes an output row. Conclude why resamplers add 2D positions (<XRef id="def-pe-2d" />) when spatial layout matters.

<SolutionBox exercise="20.2">

With positions added, the key and value in slot $j$ depend on $\mathbf v_j+\mathbf p_j$, so swapping contents while leaving positions attached to slots need not merely permute the augmented rows. An explicit scalar witness takes one latent query $q=1$, $W^Q=W^K=W^V=W^O=1$, contents $(v_1,v_2)=(0,1)$, and positions $(p_1,p_2)=(0,2)$ (with $d_k=1$). Before the swap the augmented rows are $(0,3)$, so
$$
g(0,3)=\frac{0e^0+3e^3}{e^0+e^3}\approx2.8577.
$$
After swapping only the contents they are $(1,2)$, so
$$
g(1,2)=\frac{1e^1+2e^2}{e^1+e^2}=\frac{1+2e}{1+e}\approx1.7311.
$$
The output changes. Adding 2D positions (<XRef id="def-pe-2d" />) therefore lets a resampler distinguish where a feature sits, recovering spatial sensitivity that the bag-of-patches map lacks. $\qquad\blacksquare$

</SolutionBox>

**Exercise 20.3 (Compute).** A vision encoder emits $n=576$ patch features. Compare the visual KV cache (<XRef id="prop-kv-memory" />) carried through an $N=32$-layer language model under (a) an MLP projector ($m=n$) and (b) a resampler with $m=64$. By what factor does the resampler shrink the visual cache, and how many visual tokens does it remove from each layer?

<SolutionBox exercise="20.3">

By <XRef id="prop-kv-memory" /> the visual cache scales as $\Theta(N\,m\,d)$ with $m$ the number of visual tokens. (a) Projector: $m=576$, cache $\propto 32\cdot 576\cdot d = 18432\,d$. (b) Resampler: $m=64$, cache $\propto 32\cdot 64\cdot d = 2048\,d$. The ratio is $576/64 = 9$, so the resampler shrinks the visual cache ninefold. It removes $576-64=512$ visual tokens from every one of the $32$ layers (and from each decode step), at the one-time cost $\Theta(L_r[mnd+(m+n)d^2])$, equivalently $\Theta(L_rnd(m+d))$ here because $m=O(n)$ (<XRef id="prop-resampler-cost" />). $\qquad\blacksquare$

</SolutionBox>

**Exercise 20.4 (Prove).** Given text context $C$, use the conditional Markov chain $X\to\mathbf S\to\hat Y$ given $C$ to prove $I(X;\hat Y\mid C)\le I(X;\mathbf S\mid C)$ directly from the chain rule. Identify exactly where the Markov property and nonnegativity are used, and recover the unconditional form when $C$ is fixed. (This is the engine of <XRef id="prop-connector-data-processing-bound" />.)

<SolutionBox exercise="20.4">

Expand conditional mutual information two ways:
$$
I(X;\mathbf S,\hat Y\mid C)=I(X;\mathbf S\mid C)+I(X;\hat Y\mid\mathbf S,C)=I(X;\hat Y\mid C)+I(X;\mathbf S\mid\hat Y,C).
$$
The conditional Markov property asserts $X\perp\hat Y\mid(\mathbf S,C)$, i.e. $I(X;\hat Y\mid\mathbf S,C)=0$ — the only place the Markov assumption enters. Conditional mutual information is nonnegative, so $I(X;\mathbf S\mid\hat Y,C)\ge0$ — the only place nonnegativity enters. Equating the expansions,
$$
I(X;\mathbf S\mid C)=I(X;\hat Y\mid C)+\underbrace{I(X;\mathbf S\mid\hat Y,C)}_{\ge0}\ge I(X;\hat Y\mid C).
$$
If $C$ is a fixed constant, conditioning on it changes none of these quantities, giving $I(X;\hat Y)\le I(X;\mathbf S)$. $\qquad\blacksquare$

</SolutionBox>

**Exercise 20.5 (Derive).** For the gated layer $\mathbf y=\mathbf h+\tanh(\gamma)\,c_\theta$ of <XRef id="def-gated-xattn" />, derive $\partial\mathcal L/\partial\gamma$ and $\nabla_\theta\mathcal L$ and evaluate both at $\gamma=0$, confirming <XRef id="thm-gated-identity" />. Then explain, using the same derivatives, why a *linear* gate $\gamma\,c_\theta$ (no $\tanh$) initialized at $\gamma=0$ would also start at identity but lacks the bounded-gate property of $\tanh$.

<SolutionBox exercise="20.5">

Collect $\theta\in\R^p$, let $\mathbf g_{\mathrm{vec}}=\operatorname{vec}(\partial\mathcal L/\partial\mathbf y)$, and set $\mathbf J_\theta=\partial\operatorname{vec}(c_\theta)/\partial\theta^\top$ with $\mathbf J_\theta^*=\mathbf J_\theta^\top$. The chain rule gives
$$
\frac{\partial\mathcal L}{\partial\gamma}
=\operatorname{sech}^2(\gamma)\,\inner{\mathbf g_{\mathrm{vec}}}{\operatorname{vec}(c_\theta)},
\qquad
\nabla_\theta\mathcal L
=\tanh(\gamma)\,\mathbf J_\theta^*\mathbf g_{\mathrm{vec}}.
$$
At $\gamma=0$, $\tanh 0=0$ gives $\nabla_\theta\mathcal L=\mathbf0$, while $\operatorname{sech}^2 0=1$ gives $\partial\mathcal L/\partial\gamma=\inner{\mathbf g_{\mathrm{vec}}}{\operatorname{vec}(c_\theta)}$, generically nonzero — exactly <XRef id="thm-gated-identity" />. A linear gate $\gamma c_\theta$ has the same identity start and the same gate derivative at $\gamma=0$. The downstream difference is that $|\tanh(\gamma)|<1$ bounds the scalar multiplying both $c_\theta$ and $\mathbf J_\theta^*\mathbf g_{\mathrm{vec}}$; it does not itself bound the cross-attention output or Jacobian. An unbounded linear gate places no corresponding ceiling on the contribution introduced solely by growing $\gamma$. $\qquad\blacksquare$

</SolutionBox>
