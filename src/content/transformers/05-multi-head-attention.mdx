# Multi-Head Attention

**Where we are.** The Attention chapter built a single scaled dot-product head: one triple of projections — $\mathbf W^Q, \mathbf W^K$ mapping $\R^{d}\to\R^{d_k}$ and $\mathbf W^V$ mapping $\R^{d}\to\R^{d_v}$ — producing one $n\times n$ weight matrix. One head can express only one pattern of token-to-token relationships at a time. Multi-head attention runs $h$ heads in parallel, each in its own $d_k$-dimensional subspace, then concatenates and remixes their outputs. This chapter fixes the construction (the attention head, then multi-head attention), proves the parameter count is independent of $h$ (<XRef id="prop-tf5-mha-params" />), establishes the per-head rank bound that makes a single head limited and many heads complementary (<XRef id="thm-tf5-low-rank" />), and derives the cache savings of the MQA and GQA variants (<XRef id="prop-tf5-kv-cache" />).

## The multi-head construction

A head is just the single-head attention of the previous chapter, but reading the input through its own narrow projection: instead of attending in the full $d$-dimensional space, head $i$ first compresses each token to a $d_k$-dimensional view, then runs scaled dot-product attention there.

<Theorem kind="definition" name="Attention head" id="def-tf5-attention-head">
The $i$-th **attention head** ($i = 1,\dots,h$) carries its own learned projections $\mathbf W_i^Q, \mathbf W_i^K \in \R^{d\times d_k}$ and $\mathbf W_i^V \in \R^{d\times d_v}$. Given input $\mathbf X\in\R^{n\times d}$,
$$
\operatorname{head}_i \defeq \operatorname{Attn}(\mathbf X\mathbf W_i^Q,\ \mathbf X\mathbf W_i^K,\ \mathbf X\mathbf W_i^V) \in \R^{n\times d_v},
$$
where $\operatorname{Attn}$ is scaled dot-product attention (<XRef id="def-tf4-attention" />).
</Theorem>

<MarginNote variant="formula" label="Head splitting">Each head sees $\mathbf X$ through a rank-$\le d_k$ projection, so it attends in a $d_k$-dimensional subspace.</MarginNote>

Multi-head attention is then the concatenation of all heads' outputs followed by one shared linear map back to model width. The concatenation lays the $h$ output blocks side by side within each row; the output projection lets the layer mix across heads, so the heads are not forced to occupy disjoint output coordinates.

<Theorem kind="definition" name="Multi-head attention" id="def-tf5-mha">
**Multi-head attention** concatenates the $h$ heads row-wise and applies a final linear projection:<Sidenote>The split–concatenate–project construction, with $d_k=d_v=d/h$, is the original design of <Cite key="vaswani2017attention" />.</Sidenote>
$$
\operatorname{MHA}(\mathbf X) \defeq \operatorname{Concat}(\operatorname{head}_1,\dots,\operatorname{head}_h)\,\mathbf W^O \in \R^{n\times d},
$$
where $\operatorname{Concat}$ stacks the head outputs along the feature axis (giving an $n\times h\,d_v$ matrix whose $i$-th row is $([\operatorname{head}_1]_{i,:},\dots,[\operatorname{head}_h]_{i,:})$) and $\mathbf W^O\in\R^{h\,d_v\times d}$ is the **output projection matrix**. With the standard choice $d_k = d_v = d/h$ the concatenation has width $h\cdot d_v = d$, so $\mathbf W^O\in\R^{d\times d}$.
</Theorem>

Schematically: the input $\mathbf X$ fans out to $h$ heads; each head runs attention in its $d_k$-dimensional subspace and returns an $n\times d_v$ block; the blocks concatenate into an $n\times h\,d_v$ matrix; and $\mathbf W^O$ maps that back to $n\times d$. The split into subspaces is what makes the heads independent; the output projection is what lets them recombine.

<MarginNote variant="pattern" label="Split / attend / concat / project">Four steps: project to $h$ subspaces, attend per head, concatenate, then mix with $\mathbf W^O$.</MarginNote>

## Parameter count and rank structure

The standard choice $d_k = d_v = d/h$ is not arbitrary: it is exactly the width that holds the layer's parameter count fixed as $h$ varies, so adding heads buys more independent patterns at no extra cost.

<Theorem kind="proposition" name="MHA parameter count" id="prop-tf5-mha-params">
With $d_k = d_v = d/h$, the multi-head attention layer has $4d^2$ learnable parameters (excluding biases), independent of $h$.
</Theorem>

<Theorem kind="proof">
Count the four matrix families, each step naming what it sums.

**Per head.** Each head holds $\mathbf W_i^Q,\mathbf W_i^K\in\R^{d\times d_k}$ and $\mathbf W_i^V\in\R^{d\times d_v}$. With $d_k = d_v = d/h$, each matrix has $d\cdot(d/h)$ entries, so one head has $3\,d\cdot(d/h) = 3d^2/h$ parameters.

**Across heads.** Summing over the $h$ heads, the $h$ cancels: $h\cdot 3d^2/h = 3d^2$.

**Output projection.** $\mathbf W^O\in\R^{d\times d}$ adds $d^2$.

**Total.** $3d^2 + d^2 = 4d^2$, with no residual dependence on $h$ — more heads simply means narrower heads. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="formula" label="Why d_k = d/h">Total parameters $=4d^2$ and total FLOPs match a single $d$-wide head; e.g. Llama-3 uses $d=4096$, $h=32$, $d_k=128$.</MarginNote>

We now translate each head's pre-softmax logits into a single linear-algebra object — a product of two thin matrices — whose rank is bounded by their shared inner dimension $d_k$. The correspondence is exact: *head's logit matrix* ↦ *a rank-$\le d_k$ Gram-style product*, and the bound follows from the standard rank-of-a-product inequality. Where the analogy breaks: a true Gram matrix $\mathbf W\mathbf W^\top$ is symmetric positive semidefinite, while $\mathbf W_i^Q(\mathbf W_i^K)^\top$ has two distinct learned factors and is in general neither.

<Theorem kind="theorem" name="Low-rank factorization" id="thm-tf5-low-rank">
The $n\times n$ logit matrix of head $i$ factors through the $d_k$-dimensional subspace, so its rank is at most $\min(d_k, n)$. Explicitly,
$$
\mathbf X\mathbf W_i^Q(\mathbf W_i^K)^\top\mathbf X^\top
= \mathbf X\,\underbrace{\mathbf W_i^Q(\mathbf W_i^K)^\top}_{\in\,\R^{d\times d},\ \operatorname{rank}\le d_k}\,\mathbf X^\top,
$$
and therefore $\operatorname{rank}\!\big(\text{logit matrix}_i\big)\le \min(d_k, n)$.<Sidenote>The rank bound and its expressiveness consequences are studied in <Cite key="bhojanapalli2020low" />.</Sidenote>
</Theorem>

<Theorem kind="proof">
Two steps, each naming the rank fact it uses.

**Inner product is rank-$\le d_k$.** The matrix $\mathbf W_i^Q(\mathbf W_i^K)^\top$ is the product of $\mathbf W_i^Q\in\R^{d\times d_k}$ with $(\mathbf W_i^K)^\top\in\R^{d_k\times d}$. By the rank-of-a-product inequality, $\operatorname{rank}(\mathbf A\mathbf B)\le\min(\operatorname{rank}\mathbf A,\operatorname{rank}\mathbf B)$, and each factor has rank at most $d_k$ (its smaller dimension), so
$$
\operatorname{rank}\!\big(\mathbf W_i^Q(\mathbf W_i^K)^\top\big)\le \min\!\big(\operatorname{rank}\mathbf W_i^Q,\ \operatorname{rank}\mathbf W_i^K\big)\le d_k.
$$

**Pre/post-multiplication cannot raise rank.** Left- and right-multiplying by $\mathbf X$ and $\mathbf X^\top$ only contract column and row spaces, so $\operatorname{rank}(\mathbf X\mathbf C\mathbf X^\top)\le\operatorname{rank}\mathbf C\le d_k$. Combined with the trivial bound $\operatorname{rank}\le n$ for an $n\times n$ matrix, the logit matrix of head $i$ has rank at most $\min(d_k, n)$. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="pattern" label="Rank trade-off">One head with $d_k=d$: rank up to $\min(d,n)$, one pattern. Many heads with $d_k=1$: rank-1 each, very limited. Practice keeps $d_k\ge 32$.</MarginNote>

The bound is a statement about *one head's logits before softmax*, and it explains both why a single head is limited and why several heads are complementary. A single wide head ($d_k = d$) can realize logit matrices of rank up to $\min(d, n)$ — any logit matrix only when $n \le d$ — and in any case only one pattern; whereas $h$ heads of rank $\le d_k$ give $h$ *independently parameterized* low-rank patterns — one head may track syntactic adjacency, another semantic similarity — which $\mathbf W^O$ then mixes.

<Theorem kind="remark" name="What the rank bound does not say" id="rem-tf5-low-rank">
<XRef id="thm-tf5-low-rank" /> bounds each head's logit rank by $d_k$; it does **not** assert that the full module has logit rank $h\,d_k$. The heads are concatenated *after* their separate softmax nonlinearities and then mixed by $\mathbf W^O$, so there is no single $n\times n$ logit matrix for the module to take the rank of. The "effective rank $h\,d_k$" reading is an expressiveness heuristic, not a corollary of the displayed factorization.
</Theorem>

## Efficient variants: MQA and GQA

During autoregressive decoding the keys and values of every past token are retained in a **key/value cache** so they are not recomputed at each step. In standard multi-head attention each head keeps its own cache, so cache memory grows linearly in $h$ — and at long context lengths it can exceed the model weights themselves. Multi-Query and Grouped-Query Attention cut this cost by sharing the key and value projections across heads while keeping a separate query projection per head.

<Theorem kind="definition" name="Multi-Query Attention (MQA)" id="def-tf5-mqa">
In **MQA** all $h$ heads share a single key projection $\mathbf W^K\in\R^{d\times d_k}$ and a single value projection $\mathbf W^V\in\R^{d\times d_v}$; only the query projections $\mathbf W_i^Q$ differ per head:<Sidenote>Introduced as a decoding-bandwidth optimization in <Cite key="shazeer2019mqa" />.</Sidenote>
$$
\operatorname{head}_i^{\text{MQA}} \defeq \operatorname{Attn}(\mathbf X\mathbf W_i^Q,\ \mathbf X\mathbf W^K,\ \mathbf X\mathbf W^V).
$$
</Theorem>

<Theorem kind="definition" name="Grouped-Query Attention (GQA)" id="def-tf5-gqa">
**GQA** partitions the $h$ heads into $g$ groups ($1\le g\le h$); heads within a group share one key projection and one value projection.<Sidenote>The interpolating family between MHA and MQA <Cite key="ainslie2023gqa" />.</Sidenote> MHA is the case $g = h$ (no sharing) and MQA is the case $g = 1$ (full sharing).
</Theorem>

<MarginNote variant="tip" label="Self-explanation">Before the next proof: in MHA the cache stores $\mathbf K_i,\mathbf V_i\in\R^{n\times d_k}$ for each of the $h$ heads. If $g$ groups share K and V, how many distinct such matrices remain?</MarginNote>

The cache cost follows immediately from counting distinct key/value matrices: sharing replaces "$h$ copies" by "$g$ copies," and everything else in the cache is unchanged.

<Theorem kind="proposition" name="KV cache memory" id="prop-tf5-kv-cache">
Per layer, the number of scalar entries stored in the key and value caches for a sequence of length $n$ (with $d_k = d_v$) is
$$
\text{MHA: } 2n\,h\,d_k, \qquad \text{GQA}(g): 2n\,g\,d_k, \qquad \text{MQA: } 2n\,d_k,
$$
so GQA uses a fraction $g/h$ of the MHA cache and MQA a fraction $1/h$.
</Theorem>

<Theorem kind="proof">
The cache holds one key matrix and one value matrix per *distinct* K/V projection, each of shape $n\times d_k$, hence $2n\,d_k$ entries per distinct projection. MHA has $h$ distinct projections, giving $2n\,h\,d_k$. GQA has $g$ of them, giving $2n\,g\,d_k$. MQA is GQA at $g=1$, giving $2n\,d_k$. Dividing by the MHA total yields the fractions $g/h$ and $1/h$. $\qquad\blacksquare$
</Theorem>

<MarginNote variant="formula" label="Cache savings">For $h=32$, MQA cuts KV cache $32\times$; GQA with $g=8$ keeps $1/4$ of it while staying closer to MHA quality.</MarginNote>

Two points sharpen the trade-off. First, the savings are about *bandwidth*, not chiefly parameters: the query and output projections are untouched, so the projection-parameter count moves only from $4d^2$ (MHA) to $2d^2 + 2(g/h)\,d^2$, a modest change; the real win is that fewer distinct K/V heads mean fewer cache reads and writes per generated token. Second, this is why GQA is the common middle ground — it retains most of that bandwidth gain while recovering much of MHA's quality, by reading K/V groups rather than collapsing to a single shared head <Cite key="ainslie2023gqa" />.

## Tracing the dimensions

To see the construction concretely, take $n = 3$ tokens, $d = 4$, $h = 2$, so $d_k = d_v = d/h = 2$, with input $\mathbf X\in\R^{3\times 4}$.

- **Project.** Head 1 forms $\mathbf Q_1 = \mathbf X\mathbf W_1^Q$, $\mathbf K_1 = \mathbf X\mathbf W_1^K$, $\mathbf V_1 = \mathbf X\mathbf W_1^V$, each in $\R^{3\times 2}$; head 2 likewise with its own projections.
- **Attend.** $\operatorname{head}_1 = \softmax(\mathbf Q_1\mathbf K_1^\top/\sqrt 2)\,\mathbf V_1\in\R^{3\times 2}$, and $\operatorname{head}_2\in\R^{3\times 2}$ in the same way.
- **Concatenate.** $\operatorname{Concat}(\operatorname{head}_1,\operatorname{head}_2)\in\R^{3\times 4}$, each row being $([\operatorname{head}_1]_{i,:},[\operatorname{head}_2]_{i,:})$.
- **Project out.** $\operatorname{MHA}(\mathbf X) = \operatorname{Concat}(\cdots)\,\mathbf W^O\in\R^{3\times 4}$ with $\mathbf W^O\in\R^{4\times 4}$.

The parameter count checks against <XRef id="prop-tf5-mha-params" />: the two heads contribute $2\times(3\ \text{matrices}\times 4\times 2) = 48$ and $\mathbf W^O$ contributes $4\times 4 = 16$, for $64 = 4d^2 = 4\cdot 16$. Each head attends in its own $2$-dimensional view of $\mathbf X$ — one might learn local adjacency, the other long-range similarity — and $\mathbf W^O$ blends the two perspectives into the $d$-dimensional output.

## Exercises

Attempt each before expanding its solution.

**Exercise 5.1 (Prove).** With $d_k = d_v = d/h$, show the multi-head attention layer has exactly $4d^2$ parameters (excluding biases) and that this count is independent of $h$; identify which $h$-dependence cancels.

<SolutionBox>

Take $d_k = d_v = d/h$. Each head holds $\mathbf W_i^Q,\mathbf W_i^K,\mathbf W_i^V$, each $d\cdot(d/h)$ entries, so one head has $3d^2/h$ parameters. Summing over $h$ heads the $h$ cancels: $h\cdot 3d^2/h = 3d^2$. The output projection $\mathbf W^O\in\R^{d\times d}$ adds $d^2$. Total $4d^2$, independent of $h$: each new head is correspondingly narrower ($d_k = d/h$), so per-head cost falls exactly as fast as head count rises. $\qquad\blacksquare$

</SolutionBox>

**Exercise 5.2 (Prove).** Show each head's $n\times n$ logit matrix $\mathbf Q_i\mathbf K_i^\top$ has rank at most $d_k$. Explain what this implies for a single head when $n > d_k$, and why concatenating several heads relaxes the limitation.

<SolutionBox>

For one head $\mathbf Q_i,\mathbf K_i\in\R^{n\times d_k}$, so $\mathbf L_i = \mathbf Q_i\mathbf K_i^\top\in\R^{n\times n}$. By the rank-of-a-product inequality, $\operatorname{rank}(\mathbf L_i)\le\min(\operatorname{rank}\mathbf Q_i,\operatorname{rank}\mathbf K_i)\le d_k$. When $n > d_k$ this is below the maximal rank $n$, so a single head cannot realize an arbitrary full-rank interaction. Concatenating $h$ heads supplies $h$ independent rank-$\le d_k$ patterns, which $\mathbf W^O$ mixes and later layers compose. $\qquad\blacksquare$

</SolutionBox>

**Exercise 5.3 (Derive).** Express the per-layer KV-cache size for MHA, GQA($g$), and MQA, and state each as a fraction of the MHA cache. Which variant saves cache by the factor $h$?

<SolutionBox>

The cache stores one key and one value matrix per distinct K/V projection, each $n\times d_k$, i.e. $2n\,d_k$ entries per distinct projection: MHA $2n\,h\,d_k$, GQA $2n\,g\,d_k$, MQA $2n\,d_k$. As fractions of MHA these are $1$, $g/h$, $1/h$. **MQA** saves the cache by the full factor $h$; GQA interpolates. $\qquad\blacksquare$

</SolutionBox>

**Exercise 5.4 (Compute).** For $d = 512$, $h = 8$ (so $d_k = d_v = 64$), trace the tensor shape of a batched input $(B, T, 512)$ through Q/K/V projection, the per-head split, per-head attention, concatenation, and the output projection, reporting the shape at each step.

<SolutionBox>

With $d=512$, $h=8$, head width $64$. From $(B,T,512)$: Q/K/V projection $\to (B,T,512)$; per-head split (reshape $512 = 8\times64$, move head axis) $\to (B,8,T,64)$; scores $\mathbf Q\mathbf K^\top/\sqrt{64}\to(B,8,T,T)$, softmax weights $(B,8,T,T)$, times values $\to (B,8,T,64)$; concatenate $\to (B,T,512)$; output projection $\to (B,T,512)$. The layer is shape-preserving. $\qquad\blacksquare$

</SolutionBox>

**Exercise 5.5 (Construct).** Give the GQA parameter count for $g$ groups as a function of $d$, $h$, $g$, and verify it interpolates between the MHA value at $g = h$ and the MQA value at $g = 1$.

<SolutionBox>

Query and output projections are unchanged across variants, each $d^2$. Only K,V shrink with grouping: $g$ groups give $2(g/h)d^2$. Hence $P_{\text{GQA}}(g) = d^2 + 2\tfrac{g}{h}d^2 + d^2 = 2d^2 + 2\tfrac{g}{h}d^2$. At $g=h$ this is $4d^2$ (MHA); at $g=1$ it is $2d^2 + 2d^2/h$ (MQA); affine and increasing in $g$, it interpolates monotonically. $\qquad\blacksquare$

</SolutionBox>
